🔧 Remote Management

🎯 Remote Management Neden Gerekli?

Remote Management, modern IT yönetiminin vazgeçilmez parçasıdır. Neden uzaktan yönetim önemli?

  • Zaman Tasarrufu: Fiziksel olarak sunucuya gitmeye gerek yok
  • Maliyet Tasarrufu: Seyahat maliyetleri, zaman kaybı yok
  • Hızlı Müdahale: Sorunlara anında müdahale
  • 7/24 Erişim: Her yerden, her zaman yönetim
  • Güvenlik: Şifreli bağlantılar ile güvenli erişim

🖥️ Senaryo 1: Çok Lokasyonlu Şirket Uzaktan Yönetimi

Problem: 20 lokasyonlu perakende zinciri, merkezi IT yönetimi

Çözüm: Kapsamlı remote management altyapısı

Sonuç: %80 maliyet tasarrufu, %95 sorun çözüm hızı

Remote Management Altyapısı:

# Çok lokasyonlu remote management kurulumu
# 1. RDP Gateway yapılandırması
Install-WindowsFeature -Name "RDS-Gateway" -IncludeManagementTools
Install-WindowsFeature -Name "RDS-Web-Access" -IncludeManagementTools

# 2. VPN Server yapılandırması
Install-WindowsFeature -Name "DirectAccess-VPN" -IncludeManagementTools
Install-WindowsFeature -Name "RRAS" -IncludeManagementTools

# 3. PowerShell Remoting yapılandırması
Enable-PSRemoting -Force
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*" -Force

# 4. SSH Server yapılandırması
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Start-Service -Name "sshd"
Set-Service -Name "sshd" -StartupType Automatic

# 5. Monitoring sistemi kurulumu
Install-WindowsFeature -Name "SNMP-Service" -IncludeManagementTools
Install-WindowsFeature -Name "SNMP-WMI-Provider" -IncludeManagementTools

# 6. Remote management kullanıcıları oluşturma
New-LocalUser -Name "RemoteAdmin" -Password (ConvertTo-SecureString "SecurePass123!" -AsPlainText -Force)
Add-LocalGroupMember -Group "Remote Desktop Users" -Member "RemoteAdmin"
Add-LocalGroupMember -Group "Administrators" -Member "RemoteAdmin"

🔒 Senaryo 2: Güvenli Remote Management Yapılandırması

Problem: Hassas verilerin güvenli uzaktan yönetimi

Çözüm: Multi-layer güvenlik katmanları

Sonuç: %99 güvenlik, %100 compliance

Güvenlik Yapılandırması:

# Güvenli remote management yapılandırması
# 1. RDP güvenlik ayarları
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "UserAuthentication" -Value 1
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "MinEncryptionLevel" -Value 3
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "PortNumber" -Value 3390

# 2. PowerShell Remoting güvenlik ayarları
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "192.168.1.0/24" -Force
Set-Item WSMan:\localhost\Service\Auth\Basic -Value $false
Set-Item WSMan:\localhost\Service\Auth\Certificate -Value $true

# 3. SSH güvenlik ayarları
$sshConfig = @"
Port 2222
Protocol 2
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 2
"@
Set-Content -Path "C:\ProgramData\ssh\sshd_config" -Value $sshConfig

# 4. Firewall kuralları
New-NetFirewallRule -DisplayName "RDP Custom Port" -Direction Inbound -Protocol TCP -LocalPort 3390 -Action Allow
New-NetFirewallRule -DisplayName "SSH Custom Port" -Direction Inbound -Protocol TCP -LocalPort 2222 -Action Allow
New-NetFirewallRule -DisplayName "PowerShell Remoting" -Direction Inbound -Protocol TCP -LocalPort 5985,5986 -Action Allow

# 5. Certificate-based authentication
$cert = New-SelfSignedCertificate -Subject "CN=RemoteManagement" -CertStoreLocation "Cert:\LocalMachine\My"
Set-Item WSMan:\localhost\Service\CertificateThumbprint -Value $cert.Thumbprint

📊 Senaryo 3: Otomatik Remote Management ve Monitoring

Problem: 100+ sunucunun 7/24 izlenmesi ve yönetimi

Çözüm: Otomatik monitoring ve remote management

Sonuç: %90 otomasyon, %99 uptime

Otomatik Monitoring ve Management:

# Otomatik remote management ve monitoring
# 1. Sunucu listesi
$servers = @("server1", "server2", "server3", "server4", "server5")

# 2. Otomatik health check
function Test-ServerHealth {
    param($server)
    
    $health = @{
        Server = $server
        Online = $false
        RDP = $false
        PowerShell = $false
        SSH = $false
        Services = @()
        DiskSpace = @()
        Memory = @()
    }
    
    # Ping test
    if (Test-Connection -ComputerName $server -Count 1 -Quiet) {
        $health.Online = $true
        
        # RDP test
        if (Test-NetConnection -ComputerName $server -Port 3389 -InformationLevel Quiet) {
            $health.RDP = $true
        }
        
        # PowerShell Remoting test
        try {
            $session = New-PSSession -ComputerName $server -ErrorAction Stop
            $health.PowerShell = $true
            Remove-PSSession -Session $session
        } catch {
            $health.PowerShell = $false
        }
        
        # SSH test
        if (Test-NetConnection -ComputerName $server -Port 22 -InformationLevel Quiet) {
            $health.SSH = $true
        }
        
        # Servis durumu
        try {
            $services = Invoke-Command -ComputerName $server -ScriptBlock { Get-Service | Where-Object {$_.Status -ne "Running"} }
            $health.Services = $services
        } catch {
            $health.Services = "Unable to check services"
        }
        
        # Disk alanı
        try {
            $disk = Invoke-Command -ComputerName $server -ScriptBlock { Get-WmiObject -Class Win32_LogicalDisk | Where-Object {$_.DriveType -eq 3} }
            $health.DiskSpace = $disk
        } catch {
            $health.DiskSpace = "Unable to check disk space"
        }
        
        # Memory kullanımı
        try {
            $memory = Invoke-Command -ComputerName $server -ScriptBlock { Get-WmiObject -Class Win32_OperatingSystem }
            $health.Memory = $memory
        } catch {
            $health.Memory = "Unable to check memory"
        }
    }
    
    return $health
}

# 3. Tüm sunucuları kontrol et
$allHealth = @()
foreach ($server in $servers) {
    $health = Test-ServerHealth -Server $server
    $allHealth += $health
}

# 4. Rapor oluştur
$report = $allHealth | ConvertTo-Json -Depth 3
$report | Out-File -FilePath "C:\Reports\ServerHealth-$(Get-Date -Format 'yyyy-MM-dd').json"

# 5. Alert gönder (gerekirse)
$problematicServers = $allHealth | Where-Object { -not $_.Online -or -not $_.RDP -or -not $_.PowerShell }
if ($problematicServers) {
    Send-MailMessage -To "admin@company.com" -Subject "Server Health Alert" -Body "Problematic servers: $($problematicServers.Server -join ', ')"
}

🛠️ Uzaktan Yönetim Araçları

🖥️ Remote Desktop (RDP)

Windows uzaktan masaüstü erişimi

  • Port: 3389 (TCP)
  • Protokol: RDP
  • Kullanım: Grafik arayüz erişimi
  • Güvenlik: NLA, SSL/TLS

⚡ PowerShell Remoting

Windows PowerShell uzaktan erişimi

  • Port: 5985 (HTTP), 5986 (HTTPS)
  • Protokol: WS-Management
  • Kullanım: Komut satırı yönetimi
  • Güvenlik: Kerberos, SSL/TLS

🔐 SSH (Secure Shell)

Linux/Unix uzaktan erişimi

  • Port: 22 (TCP)
  • Protokol: SSH
  • Kullanım: Komut satırı erişimi
  • Güvenlik: Public key, Password

🌐 Web-based Management

Web arayüzü üzerinden yönetim

  • Port: 80 (HTTP), 443 (HTTPS)
  • Protokol: HTTP/HTTPS
  • Kullanım: Browser tabanlı erişim
  • Güvenlik: SSL/TLS, Authentication

🖥️ Remote Desktop Yapılandırması

RDP Etkinleştirme

# PowerShell ile RDP'yi etkinleştir
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server" -Name "fDenyTSConnections" -Value 0

# RDP servisini başlat
Start-Service -Name "TermService"

# RDP servisini otomatik başlat
Set-Service -Name "TermService" -StartupType Automatic

# RDP portunu değiştir (güvenlik için)
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "PortNumber" -Value 3390

# RDP kullanıcı izinlerini ayarla
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "UserAuthentication" -Value 1

RDP Güvenlik Ayarları

# NLA (Network Level Authentication) etkinleştir
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "UserAuthentication" -Value 1

# RDP şifreleme seviyesini ayarla
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "MinEncryptionLevel" -Value 3

# RDP oturum süresini ayarla
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "MaxIdleTime" -Value 1800000

# RDP bağlantı sayısını sınırla
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name "MaxConnectionCount" -Value 2

⚡ PowerShell Remoting

PowerShell Remoting Etkinleştirme

# PowerShell Remoting'i etkinleştir
Enable-PSRemoting -Force

# Trusted hosts listesine bilgisayar ekle
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "192.168.1.100" -Force

# PowerShell Remoting servisini başlat
Start-Service -Name "WinRM"

# PowerShell Remoting servisini otomatik başlat
Set-Service -Name "WinRM" -StartupType Automatic

# PowerShell Remoting yapılandırmasını kontrol et
Get-PSSessionConfiguration

Uzaktan PowerShell Komutları

# Uzaktan PowerShell oturumu oluştur
$session = New-PSSession -ComputerName "192.168.1.100" -Credential "domain\username"

# Uzaktan komut çalıştır
Invoke-Command -Session $session -ScriptBlock { Get-Process }

# Uzaktan dosya kopyala
Copy-Item -Path "C:\local\file.txt" -Destination "C:\remote\file.txt" -ToSession $session

# Uzaktan script çalıştır
Invoke-Command -Session $session -FilePath "C:\scripts\remote-script.ps1"

# Uzaktan oturumu kapat
Remove-PSSession -Session $session

# Birden fazla bilgisayarda komut çalıştır
Invoke-Command -ComputerName "server1", "server2", "server3" -ScriptBlock { Get-Service }

Windows SSH Server

Start-Service -Name "sshd"

notepad C:\ProgramData\ssh\sshd_config

# SSH Server'ı yeniden başlat

SSH Key Yönetimi

# SSH key çifti oluştur
ssh-keygen -t rsa -b 4096 -C "user@company.com"

# Public key'i uzak sunucuya kopyala
ssh-copy-id -i ~/.ssh/id_rsa.pub user@192.168.1.100

# SSH ile bağlan
ssh user@192.168.1.100

# SSH config dosyası oluştur
New-Item -Path "$env:USERPROFILE\.ssh\config" -ItemType File
Add-Content -Path "$env:USERPROFILE\.ssh\config" -Value @"
Host server1
    HostName 192.168.1.100
    User administrator
    Port 22
    IdentityFile ~/.ssh/id_rsa
"@
