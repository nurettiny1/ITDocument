🔍 Sorun Giderme

🎯 Sorun Giderme Neden Kritik?

Sorun giderme, IT sistemlerinin sürekliliğini sağlayan en önemli beceridir. Neden sorun giderme önemli?

🔍 Sorun Giderme'nin Faydaları:

  • İş Sürekliliği: Sistem durması milyonlarca lira kayıp
  • Hızlı Çözüm: Sistematik yaklaşım ile hızlı tanı
  • Maliyet Tasarrufu: Dış destek maliyetlerinden kaçınma
  • Önleyici Bakım: Sorunları oluşmadan önce tespit
  • Müşteri Memnuniyeti: Hızlı çözüm = memnun kullanıcı

🏢 Gerçek Hayat Senaryosu:

Problem: 1000 çalışanlı şirkette sistem yavaşlığı

Sonuç: Verimlilik kaybı, çalışan memnuniyetsizliği

Çözüm: Sistematik sorun giderme ile performans artışı

🏢 Gerçek Hayat Sorun Giderme Senaryoları

🚨 Senaryo 1: Kritik Sunucu Çökmesi ve Hızlı Kurtarma

Problem: 500 çalışanlı e-ticaret firmasında web sunucusu çöktü

Çözüm: Sistematik sorun giderme ve hızlı kurtarma

Sonuç: 2 saat içinde sistem normale döndü, %99 uptime

Kritik Sunucu Çökmesi Sorun Giderme Süreci:

# Kritik sunucu çökmesi sorun giderme
# 1. İlk durum tespiti
Get-ComputerInfo | Select-Object WindowsProductName, TotalPhysicalMemory, CsProcessors
Get-Service | Where-Object {$_.Status -ne "Running"} | Select-Object Name, Status, StartType

# 2. Event log analizi
Get-WinEvent -LogName System -MaxEvents 50 | Where-Object {$_.LevelDisplayName -eq "Error"} | 
    Select-Object TimeCreated, Id, LevelDisplayName, Message | Format-Table -AutoSize

# 3. Performans analizi
Get-Counter -Counter "\Processor(_Total)\% Processor Time" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\Memory\Available MBytes" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\PhysicalDisk(_Total)\% Disk Time" -SampleInterval 1 -MaxSamples 5

# 4. Ağ bağlantısı testi
Test-NetConnection -ComputerName "google.com" -Port 80
Test-NetConnection -ComputerName "8.8.8.8" -Port 53

# 5. Servis durumu kontrolü
Get-Service | Where-Object {$_.Name -like "*IIS*" -or $_.Name -like "*Apache*" -or $_.Name -like "*Nginx*"}

# 6. Disk alanı kontrolü
Get-WmiObject -Class Win32_LogicalDisk | Where-Object {$_.DriveType -eq 3} | 
    Select-Object DeviceID, @{Name="Size(GB)";Expression={[math]::Round($_.Size/1GB,2)}}, 
    @{Name="FreeSpace(GB)";Expression={[math]::Round($_.FreeSpace/1GB,2)}}, 
    @{Name="PercentFree";Expression={[math]::Round(($_.FreeSpace/$_.Size)*100,2)}}

# 7. Yüksek kaynak kullanan işlemler
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, CPU, WorkingSet
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10 Name, WorkingSet, CPU

# 8. Otomatik kurtarma scripti
function Restart-CriticalServices {
    $criticalServices = @("IIS", "Apache", "Nginx", "MySQL", "SQL Server")
    foreach ($service in $criticalServices) {
        $svc = Get-Service -Name $service -ErrorAction SilentlyContinue
        if ($svc -and $svc.Status -ne "Running") {
            Start-Service -Name $service
            Write-Host "Started service: $service"
        }
    }
}

# 9. Sistem temizliği
Get-ChildItem -Path "C:\Windows\Temp" -Recurse | Remove-Item -Force -Recurse
Get-ChildItem -Path "C:\Temp" -Recurse | Remove-Item -Force -Recurse

# 10. Performans optimizasyonu
Get-Service | Where-Object {$_.StartType -eq "Automatic" -and $_.Status -eq "Stopped"} | 
    ForEach-Object { Start-Service -Name $_.Name }

🌐 Senaryo 2: Ağ Bağlantı Sorunları ve Çözümü

Problem: 200 çalışanlı ofiste internet bağlantısı kesintisi

Çözüm: Kapsamlı ağ tanılama ve sorun giderme

Sonuç: 1 saat içinde bağlantı normale döndü, %100 uptime

Ağ Bağlantı Sorunları Tanılama:

# Ağ bağlantı sorunları tanılama
# 1. Ağ adaptörü durumu
Get-NetAdapter | Select-Object Name, InterfaceDescription, Status, LinkSpeed
Get-NetAdapter | Where-Object {$_.Status -eq "Disconnected"} | Enable-NetAdapter

# 2. IP yapılandırması
Get-NetIPConfiguration | Select-Object InterfaceAlias, IPv4Address, IPv4DefaultGateway, DNSServer
Get-NetIPAddress | Where-Object {$_.AddressFamily -eq "IPv4"} | Select-Object IPAddress, InterfaceAlias

# 3. Ağ bağlantısı testleri
Test-NetConnection -ComputerName "8.8.8.8" -Port 53
Test-NetConnection -ComputerName "google.com" -Port 80
Test-NetConnection -ComputerName "microsoft.com" -Port 443

# 4. DNS çözümlemesi
Resolve-DnsName -Name "google.com"
Resolve-DnsName -Name "microsoft.com"
nslookup google.com

# 5. Ağ trafiği analizi
netstat -an | findstr :80
netstat -an | findstr :443
netstat -an | findstr :53

# 6. Firewall durumu
Get-NetFirewallProfile | Select-Object Name, Enabled
Get-NetFirewallRule | Where-Object {$_.Enabled -eq "True"} | Select-Object DisplayName, Direction, Action

# 7. Ağ sürücüleri
Get-NetAdapter | Get-NetAdapterDriver | Select-Object Name, DriverVersion, DriverDate

# 8. Ağ performansı
Get-Counter -Counter "\Network Interface(*)\Bytes Total/sec" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\Network Interface(*)\Packets/sec" -SampleInterval 1 -MaxSamples 5

# 9. Ağ bağlantısı yenileme
ipconfig /release
ipconfig /renew
ipconfig /flushdns

# 10. Ağ servisleri yeniden başlatma
Restart-Service -Name "Dnscache"
Restart-Service -Name "Dhcp"
Restart-Service -Name "Netman"

📧 Senaryo 3: E-posta Sistemi Sorunları ve Çözümü

Problem: 1000 çalışanlı şirkette e-posta sistemi çöktü

Çözüm: Exchange Server sorun giderme ve kurtarma

Sonuç: 3 saat içinde e-posta sistemi normale döndü, %99.9 uptime

E-posta Sistemi Sorun Giderme:

# E-posta sistemi sorun giderme
# 1. Exchange servisleri durumu
Get-Service -Name "MSExchange*" | Select-Object Name, Status, StartType
Get-Service -Name "IIS*" | Select-Object Name, Status, StartType

# 2. Exchange veritabanı durumu
Get-MailboxDatabase -Status | Select-Object Name, DatabaseSize, AvailableNewMailboxSpace, Mounted
Get-MailboxDatabase | Get-MailboxDatabaseCopyStatus | Select-Object Name, Status, CopyQueueLength, ReplayQueueLength

# 3. E-posta kuyruğu durumu
Get-Queue | Select-Object Identity, MessageCount, Status, NextHopDomain
Get-Queue | Where-Object {$_.Status -eq "Retry"} | Resume-Queue

# 4. Exchange performans sayaçları
Get-Counter -Counter "\MSExchange Database\Log Generation Checkpoint Depth" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\MSExchange Database\Log Generation Rate" -SampleInterval 1 -MaxSamples 5

# 5. Exchange log analizi
Get-EventLog -LogName "Application" -Source "MSExchange*" -After (Get-Date).AddHours(-1) | 
    Select-Object TimeGenerated, EntryType, Message | Format-Table -AutoSize

# 6. Mailbox durumu
Get-Mailbox -ResultSize Unlimited | Select-Object Name, Database, IssueWarningQuota, ProhibitSendQuota
Get-MailboxStatistics | Where-Object {$_.TotalItemSize -gt 2GB} | Select-Object DisplayName, TotalItemSize

# 7. Exchange veritabanı kurtarma
Get-MailboxDatabase | Where-Object {$_.Mounted -eq $false} | Mount-Database
Get-MailboxDatabase | Get-MailboxDatabaseCopyStatus | Where-Object {$_.Status -eq "Failed"} | 
    Update-MailboxDatabaseCopy -DeleteExistingFiles

# 8. Exchange servisleri yeniden başlatma
Restart-Service -Name "MSExchangeIS"
Restart-Service -Name "MSExchangeTransport"
Restart-Service -Name "MSExchangeRPC"

# 9. Exchange performans optimizasyonu
Get-MailboxDatabase | ForEach-Object {
    $db = $_.Name
    Write-Host "Optimizing database: $db"
    # Eseutil /d "C:\Exchange\$db\$db.edb"
}

# 10. Exchange monitoring
Get-EventLog -LogName "Application" -Source "MSExchange*" -After (Get-Date).AddHours(-1) | 
    Group-Object EntryType | Select-Object Name, Count

🔧 Sorun Giderme Metodolojisi

1. 🎯 Problemi Tanımla

  • Sorunun ne olduğunu belirle
  • Etkilenen sistemleri tespit et
  • Sorunun kapsamını değerlendir
  • Öncelik seviyesini belirle

2. 🔍 Bilgi Topla

  • Kullanıcı raporlarını analiz et
  • Sistem loglarını incele
  • Performans verilerini topla
  • Değişiklik geçmişini kontrol et

3. 🧪 Hipotez Geliştir

  • Olası nedenleri listele
  • En muhtemel nedeni belirle
  • Test stratejisi oluştur
  • Risk analizi yap

4. ⚡ Çözümü Test Et

  • Test ortamında dene
  • Adım adım uygula
  • Sonuçları değerlendir
  • Geri alma planı hazırla

5. ✅ Çözümü Uygula

  • Üretim ortamında uygula
  • Sonuçları doğrula
  • Kullanıcı onayını al
  • İzleme yap

6. 📚 Dokümante Et

  • Çözümü kaydet
  • Bilgi tabanına ekle
  • Dersleri çıkar
  • Önleyici önlemler belirle

🛠️ Tanılama Araçları

📊 Event Viewer

Windows olay günlüklerini görüntüleme

# Event Viewer'ı aç
eventvwr.msc

# PowerShell ile event log'ları görüntüle
Get-WinEvent -LogName System | Select-Object -First 10
Get-WinEvent -LogName Application | Where-Object {$_.LevelDisplayName -eq "Error"} | Select-Object -First 5

# Belirli bir olayı filtrele
Get-WinEvent -FilterHashtable @{LogName='System'; ID=1074} | Select-Object TimeCreated, Message

# Event log'ları temizle
Clear-EventLog -LogName System
Clear-EventLog -LogName Application

📈 Performance Monitor

Sistem performansını izleme

# Performance Monitor'ü aç
perfmon.msc

# PowerShell ile performans sayaçları
Get-Counter -Counter "\Processor(_Total)\% Processor Time" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\Memory\Available MBytes" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\PhysicalDisk(_Total)\% Disk Time" -SampleInterval 1 -MaxSamples 5

# Performans raporu oluştur
Get-Counter -Counter "\Processor(_Total)\% Processor Time", "\Memory\Available MBytes", "\PhysicalDisk(_Total)\% Disk Time" | Export-Counter -Path "C:\perf-report.csv" -FileFormat CSV

🔍 Resource Monitor

Kaynak kullanımını detaylı izleme

# Resource Monitor'ü aç
resmon.exe

# PowerShell ile kaynak kullanımı
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10
Get-Service | Where-Object {$_.Status -eq "Running"} | Sort-Object Name

🚨 Yaygın Sorunlar ve Çözümleri

💻 Sistem Sorunları

Sorun: Bilgisayar Yavaş Çalışıyor

  1. Task Manager'ı aç (Ctrl+Shift+Esc)
  2. Yüksek CPU/Memory kullanan işlemleri kontrol et
  3. Gereksiz programları kapat
  4. Disk temizliği yap
  5. Antivirus taraması çalıştır
  6. Startup programlarını kontrol et

Sorun: Mavi Ekran (BSOD)

  1. Hata kodunu not et
  2. Son yüklenen yazılımları kontrol et
  3. Driver güncellemelerini kontrol et
  4. System File Checker çalıştır (sfc /scannow)
  5. Memory test yap
  6. Gerekirse sistem geri yükleme yap

🌐 Ağ Sorunları

Sorun: İnternet Bağlantısı Yok

  1. Fiziksel bağlantıyı kontrol et
  2. IP adresini kontrol et (ipconfig)
  3. Gateway'e ping at
  4. DNS ayarlarını kontrol et
  5. Router'ı yeniden başlat
  6. Network adapter'ı yeniden başlat

Sorun: RDP Bağlantısı Kurulamıyor

  1. RDP servisinin çalıştığını kontrol et
  2. Firewall kurallarını kontrol et
  3. Kullanıcı izinlerini kontrol et
  4. Port ayarlarını kontrol et
  5. Network connectivity'yi test et

📊 Log Analizi

Windows Event Logs

# Sistem loglarını analiz et
Get-WinEvent -LogName System | Where-Object {$_.LevelDisplayName -eq "Error"} | Select-Object TimeCreated, Id, LevelDisplayName, Message | Format-Table -AutoSize

# Uygulama loglarını analiz et
Get-WinEvent -LogName Application | Where-Object {$_.LevelDisplayName -eq "Error"} | Select-Object TimeCreated, Id, LevelDisplayName, Message | Format-Table -AutoSize

# Güvenlik loglarını analiz et
Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4625} | Select-Object TimeCreated, Message

# Belirli bir zaman aralığındaki logları analiz et
$StartTime = (Get-Date).AddDays(-1)
$EndTime = Get-Date
Get-WinEvent -FilterHashtable @{LogName='System'; StartTime=$StartTime; EndTime=$EndTime} | Where-Object {$_.LevelDisplayName -eq "Error"}

IIS Logs

# IIS log dosyalarını analiz et
$LogPath = "C:\inetpub\logs\LogFiles\W3SVC1"
Get-ChildItem -Path $LogPath -Filter "*.log" | Sort-Object LastWriteTime -Descending | Select-Object -First 1 | ForEach-Object {
    Get-Content $_.FullName | Select-String "404" | Select-Object -First 10
}

# HTTP status kodlarını analiz et
$LogContent = Get-Content "C:\inetpub\logs\LogFiles\W3SVC1\ex*.log" | Select-String "GET"
$LogContent | ForEach-Object {
    $_.Line -split " " | Select-Object -Index 8
} | Group-Object | Sort-Object Count -Descending

⚡ Performans Sorun Giderme

CPU Kullanımı

# Yüksek CPU kullanan işlemleri bul
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, CPU, WorkingSet

# CPU kullanımını sürekli izle
while ($true) {
    Get-Counter -Counter "\Processor(_Total)\% Processor Time" -SampleInterval 1 -MaxSamples 1
    Start-Sleep -Seconds 5
}

# Belirli bir işlemin CPU kullanımını izle
Get-Process -Name "chrome" | ForEach-Object {
    Get-Counter -Counter "\Process($($_.ProcessName))\% Processor Time" -SampleInterval 1 -MaxSamples 5
}

Memory Kullanımı

# Memory kullanımını kontrol et
Get-Counter -Counter "\Memory\Available MBytes" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\Memory\% Committed Bytes In Use" -SampleInterval 1 -MaxSamples 5

# Yüksek memory kullanan işlemleri bul
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10 Name, WorkingSet, CPU

# Memory kullanımını sürekli izle
while ($true) {
    $Memory = Get-Counter -Counter "\Memory\Available MBytes" -SampleInterval 1 -MaxSamples 1
    Write-Host "Available Memory: $($Memory.CounterSamples[0].CookedValue) MB"
    Start-Sleep -Seconds 5
}

Disk Kullanımı

# Disk kullanımını kontrol et
Get-WmiObject -Class Win32_LogicalDisk | Select-Object DeviceID, @{Name="Size(GB)";Expression={[math]::Round($_.Size/1GB,2)}}, @{Name="FreeSpace(GB)";Expression={[math]::Round($_.FreeSpace/1GB,2)}}, @{Name="PercentFree";Expression={[math]::Round(($_.FreeSpace/$_.Size)*100,2)}}

# Disk performansını izle
Get-Counter -Counter "\PhysicalDisk(_Total)\% Disk Time" -SampleInterval 1 -MaxSamples 5
Get-Counter -Counter "\PhysicalDisk(_Total)\Avg. Disk Queue Length" -SampleInterval 1 -MaxSamples 5

# En büyük dosyaları bul
Get-ChildItem -Path "C:\" -Recurse -File | Sort-Object Length -Descending | Select-Object -First 20 Name, @{Name="Size(GB)";Expression={[math]::Round($_.Length/1GB,2)}}, FullName
