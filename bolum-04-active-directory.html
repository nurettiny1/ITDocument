👥 Active Directory (AD DS)

🎯 Active Directory Neden Gerekli?

Active Directory, büyük organizasyonlarda kaosu önlemek için tasarlanmıştır. Neden Active Directory kullanmalıyız?

🔍 Active Directory'nin Faydaları:

  • Merkezi Yönetim: 1000+ kullanıcıyı tek yerden yönetebilirsiniz
  • Güvenlik: Tüm erişimleri kontrol edebilirsiniz
  • Otomasyon: Tekrarlayan işlemleri otomatikleştirebilirsiniz
  • Ölçeklenebilirlik: Şirket büyüdükçe sistem büyür
  • Maliyet Tasarrufu: Manuel işlemler yerine otomasyon

🏢 Gerçek Hayat Senaryosu:

Problem: 500 çalışanlı şirkette her departman kendi kullanıcılarını yönetiyor

Sonuç: Kaos, güvenlik açıkları, maliyet artışı

Çözüm: Active Directory ile merkezi yönetim

📚 Temel Kavramlar

🌐 Domain

Bir ağdaki bilgisayarların ve kaynakların mantıksal gruplandırması

  • Merkezi yönetim
  • Güvenlik sınırları
  • DNS tabanlı isimlendirme
  • Örnek: company.local

🏢 Domain Controller (DC)

Active Directory veritabanını barındıran sunucu

  • Kullanıcı kimlik doğrulama
  • Veri replikasyonu
  • Güvenlik politikaları
  • DNS servisleri

👤 User Account

Kullanıcıların sisteme erişimini sağlayan hesap

  • Kimlik doğrulama
  • Yetkilendirme
  • Profil yönetimi
  • Grup üyelikleri

👥 Group

Kullanıcıları ve kaynakları gruplandırmak için kullanılan yapı

  • Yetki yönetimi
  • E-posta dağıtım listeleri
  • Politika uygulama
  • Kaynak erişimi

3. 🤖 Unattended Kurulum (Otomatik)

Neden Unattended kullanmalıyız? Toplu kurulum, otomasyon

🚀 Otomatik Kurulum Scripti
# Otomatik AD kurulum scripti
function Install-ADAutomated {
    param(
        [string]$DomainName,
        [string]$SafeModePassword,
        [string]$ForestMode = "Win2022",
        [string]$DomainMode = "Win2022"
    )
    
    try {
        # AD DS rolü ekle
        Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
        
        # Domain Controller kur
        Install-ADDSForest -DomainName $DomainName `
            -SafeModeAdministratorPassword (ConvertTo-SecureString $SafeModePassword -AsPlainText -Force) `
            -ForestMode $ForestMode `
            -DomainMode $DomainMode `
            -Force
        
        Write-Host "AD kurulumu başarılı: $DomainName" -ForegroundColor Green
    }
    catch {
        Write-Error "AD kurulumu başarısız: $($_.Exception.Message)"
    }
}

# Script kullanımı
Install-ADAutomated -DomainName "company.local" -SafeModePassword "Password123!"

🤖 PowerShell Script ile (Otomatik)
# Toplu kullanıcı oluşturma scripti
function New-BulkUsers {
    param(
        [string]$CSVPath = "C:\Users.csv"
    )
    
    $users = Import-Csv -Path $CSVPath
    
    foreach ($user in $users) {
        try {
            New-ADUser -SamAccountName $user.Username `
                -UserPrincipalName "$($user.Username)@company.local" `
                -Name $user.FullName `
                -GivenName $user.FirstName `
                -Surname $user.LastName `
                -DisplayName $user.FullName `
                -Path "OU=Users,DC=company,DC=local" `
                -AccountPassword (ConvertTo-SecureString $user.Password -AsPlainText -Force) `
                -Enabled $true
            
            # Departman grubuna ekleme
            Add-ADGroupMember -Identity $user.Department -Members $user.Username
            
            Write-Host "Kullanıcı oluşturuldu: $($user.FullName)" -ForegroundColor Green
        }
        catch {
            Write-Error "Kullanıcı oluşturulamadı: $($user.FullName) - $($_.Exception.Message)"
        }
    }
}

# CSV dosyası örneği
# Username,FirstName,LastName,FullName,Password,Department
# john.doe,John,Doe,John Doe,TempPass123!,IT
# jane.smith,Jane,Smith,Jane Smith,TempPass123!,HR

# Script çalıştırma
New-BulkUsers -CSVPath "C:\Users.csv"

Toplu Kullanıcı Oluşturma

# Grup üyelerini listeleme

📧 Distribution Groups

E-posta dağıtım listeleri için kullanılır

# Distribution group oluşturma
New-ADGroup -Name "All Employees" `
    -GroupCategory Distribution `
    -GroupScope Universal `
    -Description "All company employees"

# E-posta adresi atama
Set-ADGroup -Identity "All Employees" -Add @{mail="all@company.local"}

Grup Kapsamları (Group Scopes)

🏠 Domain Local

  • Sadece aynı domain içinde kullanılabilir
  • Kaynak izinleri için ideal
  • Diğer domain'lerden üye alabilir

🌐 Global

  • Forest içinde her yerde kullanılabilir
  • Kullanıcı grupları için ideal
  • Sadece aynı domain'den üye alabilir

🌍 Universal

  • Forest içinde her yerde kullanılabilir
  • Her domain'den üye alabilir
  • En esnek grup türü

📁 Organizational Units (OU)

OU Oluşturma ve Yönetimi

# OU oluşturma
New-ADOrganizationalUnit -Name "IT Department" `
    -Path "DC=company,DC=local" `

    -Path "OU=IT Department,DC=company,DC=local"

# Kullanıcıyı OU'ya taşıma
Move-ADObject -Identity "CN=john.doe,CN=Users,DC=company,DC=local" `

🔧 AD Troubleshooting Rehberi

❌ Yaygın AD Sorunları ve Çözümleri

Problem 1: Kullanıcı Login Olamıyor

Semptomlar: "The user name or password is incorrect" hatası

Olası Nedenler:

  • Yanlış şifre
  • Hesap kilitli
  • Domain controller erişim sorunu
  • DNS çözümleme hatası
Çözüm Adımları:
# 1. Kullanıcı hesap durumunu kontrol et
Get-ADUser -Identity "john.doe" -Properties LockedOut, Enabled, PasswordExpired

# 2. Hesap kilidini aç
Unlock-ADAccount -Identity "john.doe"

# 3. DNS çözümlemesini test et
nslookup contoso.com
Test-NetConnection -ComputerName "DC01.contoso.com" -Port 389

# 4. Domain controller erişimini test et
Test-ComputerSecureChannel -Verbose

# 5. Kullanıcı şifresini sıfırla
Set-ADAccountPassword -Identity "john.doe" -NewPassword (ConvertTo-SecureString "NewPassword123!" -AsPlainText -Force)

Problem 2: GPO Uygulanmıyor

Semptomlar: Group Policy ayarları uygulanmıyor

Olası Nedenler:

  • GPO link edilmemiş
  • Security filtering yanlış
  • WMI filter sorunu
  • Replication sorunu
Çözüm Adımları:
# 1. GPO durumunu kontrol et
Get-GPO -Name "Security Policy" | Get-GPOReport -ReportType Html

# 2. GPO link durumunu kontrol et
Get-GPO -Name "Security Policy" | Get-GPLink

# 3. Security filtering kontrol et
Get-GPPermission -Name "Security Policy" -TargetName "Domain Users" -TargetType Group

# 4. GPO'yu zorla uygula
gpupdate /force

# 5. Replication durumunu kontrol et
repadmin /showrepl
repadmin /syncall

Problem 3: Replication Sorunları

Semptomlar: Değişiklikler diğer DC'lere replike olmuyor

Olası Nedenler:

  • Network bağlantı sorunu
  • DNS çözümleme hatası
  • Firewall engeli
  • Kerberos authentication sorunu
Çözüm Adımları:
# 1. Replication durumunu kontrol et
repadmin /showrepl
repadmin /replsummary

# 2. Replication'ı zorla başlat
repadmin /syncall /AdeP

# 3. DNS kayıtlarını kontrol et
nslookup _ldap._tcp.dc._msdcs.contoso.com

# 4. Kerberos ticket'larını kontrol et
klist tickets

# 5. Replication metadata'yı temizle
repadmin /removelingeringobjects DC01 DC02
