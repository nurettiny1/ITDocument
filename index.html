IT Eğitim Dokümanı

Bölüm 1-2: Temel IT Kavramları, Windows Server Yönetimi ve İleri Seviye Konular

1. Giriş

IT Departmanının Görevi

Bilgi Teknolojileri (IT) departmanı, kurumsal bilgi sistemlerinin planlanması, kurulumu, yönetimi ve bakımından sorumludur. Bu departman, şirketin teknolojik altyapısının güvenli, verimli ve sürekli çalışmasını sağlar.

IT Departmanının Temel Sorumlulukları:

  • Sistem Yönetimi: Sunucu, ağ ve veritabanı sistemlerinin kurulumu ve yönetimi
  • Güvenlik: Siber güvenlik önlemlerinin alınması ve sürdürülmesi
  • Teknik Destek: Kullanıcıların teknik sorunlarının çözülmesi
  • Yedekleme ve Kurtarma: Veri güvenliğinin sağlanması
  • Yazılım Yönetimi: Lisanslı yazılımların dağıtımı ve güncellenmesi
  • Proje Yönetimi: IT projelerinin planlanması ve uygulanması

Sistem Uzmanlarının Rolü

Sistem uzmanları, IT departmanının kalbidir ve kurumsal altyapının teknik yönetiminden sorumludur.

Sistem Uzmanının Temel Görevleri:

  • Sunucu Yönetimi: Windows Server, Linux sunucuların kurulumu ve yönetimi
  • Ağ Yönetimi: LAN, WAN, VPN yapılarının kurulumu ve yönetimi
  • Active Directory: Kullanıcı hesapları, gruplar ve politikaların yönetimi
  • Güvenlik: Firewall, antivirüs ve güvenlik politikalarının uygulanması
  • Performans İzleme: Sistem performansının izlenmesi ve optimizasyonu
  • Yedekleme Stratejileri: Veri yedekleme ve kurtarma planlarının uygulanması
💡 Not: Modern sistem uzmanları, bulut teknolojileri (Azure, AWS) ve DevOps pratikleri konusunda da bilgi sahibi olmalıdır.

Server Rollerinin Eklenmesi

Active Directory Domain Services (AD DS)

🖥️ Grafik Arayüzü ile AD DS Kurulumu
1 Server Manager'da Rol Ekleme

Neden: Active Directory, Windows Server'da bir "rol" olarak çalışır. Bu rolü eklemeden AD kullanamazsınız.

  • Server Manager → "Add roles and features": Server Manager, Windows Server'ın ana yönetim konsoludur. Buradan tüm roller ve özellikler yönetilir.
  • "Role-based or feature-based installation": Bu seçenek, sunucuya yeni işlevsellik eklemek için kullanılır. Role-based, sunucunun belirli bir görevi (AD, DNS, DHCP) üstlenmesini sağlar.
  • Sunucunuzu seçin: Birden fazla sunucu yönetiyorsanız, hangi sunucuya rol ekleyeceğinizi belirtmeniz gerekir.
  • "Active Directory Domain Services" işaretleyin: Bu, sunucunuzu bir Domain Controller yapacak temel rolüdür. İşaretlediğinizde otomatik olarak gerekli bağımlılıklar da eklenir.
  • Gerekli features'ları da ekleyin: AD DS rolü, bazı ek özellikler gerektirir (Group Policy Management, AD DS Tools vb.). Bunlar otomatik olarak eklenir.
2 AD DS Forest Kurulumu

Neden: Rol eklendikten sonra, sunucuyu gerçek bir Domain Controller'a dönüştürmeniz gerekir. Bu işlem "promotion" olarak adlandırılır.

  • Kurulum tamamlandıktan sonra Server Manager'da sarı uyarı işareti görünür: Bu, rolün eklendiğini ama henüz yapılandırılmadığını gösterir. Sarı uyarı, "bu rolü yapılandırmanız gerekiyor" anlamına gelir.
  • "Promote this server to a domain controller" tıklayın: Bu, sunucunuzu gerçek bir Domain Controller'a dönüştürecek sihirbazı başlatır.
  • Deployment Configuration: "Add a new forest" seçin: Forest, Active Directory'nin en üst seviye mantıksal yapısıdır. Yeni bir forest oluşturuyorsunuz çünkü ilk Domain Controller'ınız.
  • Root domain name: "contoso.local" yazın: Bu, domain'inizin adıdır. ".local" uzantısı, iç ağ için kullanılır. Gerçek şirket adınızı kullanabilirsiniz (örn: "sirketim.com").
  • Forest functional level: "Windows Server 2016" seçin: Bu, forest'unuzun hangi Windows Server sürümünün özelliklerini kullanabileceğini belirler. Daha yüksek seviye, daha fazla özellik demektir.
3 Domain Controller Seçenekleri

Neden: Domain Controller'ınızın hangi ek hizmetleri sağlayacağını belirlemeniz gerekir. Bu seçenekler, sunucunuzun yeteneklerini artırır.

  • Domain Controller Options: Bu bölümde, Domain Controller'ınızın ek özelliklerini seçersiniz.
  • ☑️ "Domain Name System (DNS) server" işaretleyin: DNS, domain adlarını IP adreslerine çevirir. AD ile birlikte çalışması gerekir. İşaretlemezseniz, ayrı bir DNS sunucusu kurmanız gerekir.
  • ☑️ "Global Catalog (GC)" işaretleyin: Global Catalog, tüm domain'lerdeki nesnelerin özet bilgilerini tutar. İlk Domain Controller'da mutlaka olmalıdır. Kullanıcı girişlerini hızlandırır.
  • DSRM Password: Güçlü bir şifre belirleyin: Directory Services Restore Mode şifresi, sunucu sorunlu olduğunda kurtarma modunda giriş yapmak için kullanılır. Bu şifreyi kaybetmeyin!
  • Additional Options: Varsayılan ayarları kabul edin: Bu ayarlar genellikle doğru yapılandırılmıştır. Değiştirmenize gerek yoktur.
4 Kurulum Tamamlama

Neden: Son kontrolleri yapıp kurulumu başlatmadan önce, her şeyin doğru olduğundan emin olmanız gerekir.

  • Review Options: Ayarları kontrol edin: Bu ekranda, yaptığınız tüm seçimlerin özetini görürsünüz. Domain adı, forest seviyesi, DNS ayarları vb. Her şeyin doğru olduğundan emin olun.
  • Prerequisites Check: Hataları kontrol edin: Sistem, kurulum için gerekli koşulları kontrol eder. Eğer hata varsa (örneğin, yeterli disk alanı yok), burada görünür. Hataları düzeltmeden devam edemezsiniz.
  • Install: Kurulumu başlatın: Tüm kontroller başarılıysa, kurulum başlar. Bu işlem 10-30 dakika sürebilir, sabırlı olun.
  • Sunucu otomatik olarak yeniden başlayacak: Kurulum tamamlandıktan sonra, sunucu yeniden başlar ve artık bir Domain Controller olarak çalışır. Giriş yaparken domain hesabı kullanabilirsiniz.

DNS Server

🖥️ Grafik Arayüzü ile DNS Kurulumu
1 DNS Server Rolü Ekleme

Neden: DNS (Domain Name System), domain adlarını IP adreslerine çevirir. Active Directory ile birlikte çalışması gerekir. Örneğin, "server1.contoso.local" adresini 192.168.1.10 IP'sine çevirir.

  • Server Manager → "Add roles and features": DNS Server da bir rol olarak eklenir. Bu, sunucunuzun DNS hizmeti sağlamasını sağlar.
  • "Role-based or feature-based installation" seçin: DNS Server rolü, sunucunuzun domain adı çözümlemesi yapmasını sağlar.
  • Sunucunuzu seçin: Hangi sunucuya DNS rolü ekleyeceğinizi belirtin. Genellikle Domain Controller ile aynı sunucuya eklenir.
  • "DNS Server" işaretleyin: Bu rol, sunucunuzun DNS sorgularını yanıtlamasını sağlar. AD ile entegre çalışır.
  • Gerekli features'ları da ekleyin: DNS Server rolü, bazı ek araçlar gerektirir. Bunlar otomatik olarak eklenir.
2 DNS Manager ile Zone Oluşturma

Neden: DNS Zone, belirli bir domain için DNS kayıtlarını tutan bir veritabanıdır. Örneğin, "contoso.local" domain'i için bir zone oluşturmanız gerekir.

  • Start → "DNS" yazın ve DNS Manager'ı açın: DNS Manager, DNS sunucunuzu yönetmek için kullanılan grafik arayüzdür. Buradan zone'ları, kayıtları yönetebilirsiniz.
  • Sol panelde sunucunuzu genişletin: DNS Manager açıldığında, sol panelde sunucunuzu göreceksiniz. Genişlettiğinizde "Forward Lookup Zones" ve "Reverse Lookup Zones" klasörlerini görürsünüz.
  • "Forward Lookup Zones" üzerine sağ tıklayın: Forward Lookup Zone, domain adını IP adresine çevirir (örn: server1.contoso.local → 192.168.1.10). Reverse Lookup Zone ise tam tersini yapar.

5. DNS & DHCP Yönetimi

DNS Kayıt Türleri

DNS (Domain Name System), domain isimlerini IP adreslerine çeviren sistemdir. Farklı kayıt türleri farklı amaçlara hizmet eder.

Temel DNS Kayıt Türleri:

Kayıt Türü Açıklama Örnek Kullanım Amacı
A IPv4 adres kaydı www.contoso.com → 192.168.1.10 Domain'i IPv4 adresine yönlendirme
AAAA IPv6 adres kaydı www.contoso.com → 2001:db8::1 Domain'i IPv6 adresine yönlendirme
CNAME Canonical Name (Alias) mail.contoso.com → mailserver.contoso.com Bir domain'i başka bir domain'e yönlendirme
MX Mail Exchange contoso.com → mail.contoso.com (Priority: 10) E-posta sunucusu belirtme
NS Name Server contoso.com → ns1.contoso.com Domain için yetkili DNS sunucuları
PTR Pointer (Reverse DNS) 192.168.1.10 → www.contoso.com IP adresini domain'e çevirme
TXT Text Record contoso.com → "v=spf1 include:_spf.google.com ~all" Metin bilgileri (SPF, DKIM vb.)

Get-DnsServerResourceRecord -ZoneName "contoso.local" -RRType A

DHCP Scope ve Reservation Ayarları

DHCP Scope Nedir?

DHCP Scope, DHCP sunucusunun istemcilere dağıtabileceği IP adresi aralığıdır. Her scope belirli bir ağ segmenti için tanımlanır.

DHCP Scope Oluşturma

# DHCP sunucusu kurulumu
Install-WindowsFeature -Name DHCP -IncludeManagementTools

# DHCP yetkilendirme
Add-DhcpServerInDC -DnsName "dc01.contoso.local" -IPAddress "192.168.1.10"

# Yeni scope oluşturma
Add-DhcpServerv4Scope -Name "IT_Department" -StartRange "192.168.1.100" -EndRange "192.168.1.200" -SubnetMask "255.255.255.0" -State Active

# Scope seçeneklerini yapılandırma
Set-DhcpServerv4OptionValue -ScopeId "192.168.1.0" -DnsServer "192.168.1.10","8.8.8.8" -DnsDomain "contoso.local"
Set-DhcpServerv4OptionValue -ScopeId "192.168.1.0" -Router "192.168.1.1"

# Scope bilgilerini görüntüleme
Get-DhcpServerv4Scope

# Belirli scope'un detaylarını görüntüleme
Get-DhcpServerv4Scope -ScopeId "192.168.1.0"

DHCP Reservation (Rezervasyon)

DHCP Reservation, belirli bir MAC adresine sabit IP adresi atanmasını sağlar. Bu sayede sunucular ve yazıcılar gibi cihazlar her zaman aynı IP adresini alır.

# MAC adresini öğrenme (client'ta çalıştırın)
Get-NetAdapter | Select-Object Name, MacAddress

# Reservation oluşturma
Add-DhcpServerv4Reservation -ScopeId "192.168.1.0" -IPAddress "192.168.1.50" -ClientId "00-15-5D-01-02-03" -Name "FileServer"

# Reservation'ları listeleme
Get-DhcpServerv4Reservation -ScopeId "192.168.1.0"

# Reservation'ı silme
Remove-DhcpServerv4Reservation -ScopeId "192.168.1.0" -IPAddress "192.168.1.50"

# Scope'u etkinleştirme/devre dışı bırakma

Set-DhcpServerv4Scope -ScopeId "192.168.1.0" -State InActive

Get-DhcpServerv4ScopeStatistics -ScopeId "192.168.1.0"

NTFS vs Share İzinleri

Windows Server'da dosya ve klasör erişimini kontrol etmek için iki tür izin sistemi kullanılır: NTFS izinleri ve Share izinleri.

NTFS İzinleri

NTFS izinleri, dosya sistemi seviyesinde çalışır ve hem yerel hem de ağ erişimini kontrol eder.

İzin Türü Açıklama Kullanım
Full Control Tüm izinleri içerir Yöneticiler için
Modify Okuma, yazma, silme, çalıştırma Kullanıcılar için
Read & Execute Okuma ve çalıştırma Uygulama dosyaları için
List Folder Contents Klasör içeriğini listeleme Klasörler için
Read Sadece okuma Referans dosyaları için
Write Sadece yazma Geçici klasörler için

Share İzinleri

Share izinleri, ağ üzerinden erişim sırasında uygulanır ve sadece ağ erişimini kontrol eder.

İzin Türü Açıklama Kullanım
Full Control Tüm izinleri içerir Yöneticiler için
Change Okuma, yazma, silme Kullanıcılar için
Read Sadece okuma Genel erişim için

İzin Yönetimi PowerShell Komutları

# NTFS izinlerini görüntüleme
Get-Acl "C:\SharedFolder" | Format-List

# NTFS izni ekleme
$acl = Get-Acl "C:\SharedFolder"
$accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule("contoso\ayilmaz", "FullControl", "ContainerInherit,ObjectInherit", "None", "Allow")
$acl.SetAccessRule($accessRule)
Set-Acl "C:\SharedFolder" $acl

# NTFS iznini kaldırma
$acl = Get-Acl "C:\SharedFolder"
$accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule("contoso\ayilmaz", "FullControl", "ContainerInherit,ObjectInherit", "None", "Allow")
$acl.RemoveAccessRule($accessRule)
Set-Acl "C:\SharedFolder" $acl

# Share oluşturma
New-SmbShare -Name "IT_Documents" -Path "C:\IT_Documents" -FullAccess "contoso\IT_Users"

# Share izinlerini görüntüleme
Get-SmbShareAccess -Name "IT_Documents"

# Share iznini değiştirme
Grant-SmbShareAccess -Name "IT_Documents" -AccountName "contoso\ayilmaz" -AccessRight Full -Force
⚠️ Önemli: Gerçek izin, NTFS ve Share izinlerinin en kısıtlayıcı olanıdır. Her iki izin türünü de doğru yapılandırdığınızdan emin olun.

Email Security (E-posta Güvenliği)

E-posta güvenliği, spam, phishing ve malware saldırılarından korunma yöntemleridir.

📧 SPF, DKIM, DMARC Yapılandırması

1 SPF (Sender Policy Framework) Kaydı

Neden: SPF, e-posta gönderen sunucunun yetkili olup olmadığını kontrol eder. Sahte e-posta gönderimini engeller.

  • DNS yönetim paneline giriş yapın: Domain sağlayıcınızın DNS ayarlarına erişin
  • TXT kaydı oluşturun: "v=spf1 include:_spf.google.com ~all"
  • IP adreslerini ekleyin: "v=spf1 ip4:192.168.1.10 include:_spf.google.com ~all"
  • Test edin: SPF kaydınızı online araçlarla test edin
2 DKIM (DomainKeys Identified Mail) Yapılandırması

Neden: DKIM, e-postaların gerçekten gönderen domain'den geldiğini doğrular. E-posta bütünlüğünü sağlar.

  • Mail sunucunuzda DKIM'i etkinleştirin: Exchange Server veya diğer mail sunucularında
  • Public key'i DNS'e ekleyin: CNAME kaydı olarak
  • Selector'ı belirleyin: Örneğin "selector1._domainkey"
  • Test edin: DKIM imzasını kontrol edin
3 DMARC (Domain-based Message Authentication) Politikası

Neden: DMARC, SPF ve DKIM sonuçlarına göre e-postaların ne yapılacağını belirler. Phishing saldırılarını engeller.

  • DMARC politikası oluşturun: "v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com"
  • Raporlama e-postası ayarlayın: DMARC raporlarını alacak e-posta adresi
  • Politika seviyesini belirleyin: none, quarantine, reject
  • Raporları analiz edin: DMARC raporlarını düzenli olarak kontrol edin

            TotalDisk = [math]::Round(($disk | Measure-Object Size -Sum).Sum / 1GB, 2)
            LastLogon = $computer.LastLogonDate
        }
        $inventory += $computerInfo
    }
    catch {
        Write-Warning "Bilgisayara erişilemedi: $($computer.Name)"
    }
}

Neden: İkinci bir sunucuyu Domain Controller yapmak için önce AD DS rolünü eklemeniz gerekir. Bu, sunucunun AD hizmeti sağlamasını sağlar.

Neden: Rol eklendikten sonra, sunucuyu gerçek bir Domain Controller'a dönüştürmeniz gerekir. Bu işlem, sunucuyu mevcut domain'e ekler.

🖥️ Grafik Arayüzü ile FSMO Rolleri Yönetimi

1 FSMO Rollerini Görüntüleme

Neden: FSMO rolleri, AD'de kritik işlevleri yerine getirir. Hangi sunucuda hangi rollerin olduğunu bilmek önemlidir.

  • Active Directory Users and Computers açın: Start → "dsa.msc" yazın
  • Domain'e sağ tıklayın: contoso.local
  • "Operations Masters" seçin: FSMO rolleri görüntüleme
  • RID, PDC, Infrastructure tab'larını kontrol edin: Her tab farklı rolleri gösterir
  • Current Operations Master'ı not edin: Hangi sunucuda olduğunu kaydedin
2 FSMO Rol Transferi

Neden: FSMO rolleri, sunucu bakımı veya yük dağılımı için başka sunuculara transfer edilebilir.

  • Operations Masters penceresinde "Change" tıklayın: Rol transferi başlatır
  • Hedef sunucuyu seçin: Rolü alacak sunucu
  • "Yes" tıklayın: Transferi onaylar
  • Transfer durumunu kontrol edin: Başarılı olup olmadığını kontrol edin
  • Diğer rolleri de transfer edin: Gerekirse
3 Schema Master ve Domain Naming Master

Neden: Bu roller, Forest seviyesindedir ve farklı araçlarla yönetilir.

  • Schema Master için: Start → "regsvr32 schmmgmt.dll" çalıştırın
  • MMC açın: Start → "mmc" yazın
  • Add/Remove Snap-in: File → Add/Remove Snap-in
  • Active Directory Schema ekleyin: Schema Master rolü için
  • Active Directory Domains and Trusts ekleyin: Domain Naming Master için

FSMO Rolleri:

Rol Açıklama Kapsam
Schema Master Schema değişiklikleri Forest
Domain Naming Master Domain ekleme/silme Forest
PDC Emulator Zaman senkronizasyonu, şifre değişiklikleri Domain
RID Master RID havuzu yönetimi Domain
Infrastructure Master Cross-domain referansları Domain

AD Sites & Services

AD Sites & Services, Active Directory replikasyonunu ve hizmet konumlandırmasını yönetir.

🖥️ Grafik Arayüzü ile AD Sites & Services Yönetimi

1 AD Sites and Services Açma

Neden: AD Sites & Services, replikasyon topolojisini görsel olarak yönetmenizi sağlar. Bu, ağ yapınıza göre en uygun replikasyon stratejisini belirlemenize yardımcı olur.

  • Start → "dssite.msc" yazın: AD Sites and Services'ı doğrudan açar
  • Veya Server Manager → Tools → "Active Directory Sites and Services": Alternatif yöntem
  • Sol panelde "Sites" genişletin: Mevcut siteleri görüntüler

2 Yeni Site Oluşturma

Neden: Farklı lokasyonlardaki sunucular için ayrı siteler oluşturarak, replikasyon trafiğini optimize edebilirsiniz.

  • Sites klasörüne sağ tıklayın: "New Site" seçin
  • Site adı girin: Örnek: "Istanbul-Site"
  • Site link seçin: DEFAULTIPSITELINK (varsayılan)
  • OK tıklayın: Site oluşturulur
  • Subnet oluşturmak için "Subnets" klasörüne sağ tıklayın: "New Subnet" seçin
3 Subnet Yapılandırması

Neden: Subnet'ler, hangi IP aralığının hangi siteye ait olduğunu belirler. Bu, kullanıcıların en yakın Domain Controller'a yönlendirilmesini sağlar.

  • Subnet adı girin: Örnek: "192.168.1.0/24"
  • Site seçin: Oluşturduğunuz site'ı seçin
  • Description ekleyin: İsteğe bağlı açıklama
  • OK tıklayın: Subnet oluşturulur
4 Site Link Yapılandırması

Neden: Site link'ler, siteler arası replikasyon maliyetini ve sıklığını belirler. Düşük maliyet, öncelikli replikasyon anlamına gelir.

  • Inter-Site Transports → IP klasörünü genişletin: Site link'leri görüntüler
  • DEFAULTIPSITELINK'e sağ tıklayın: "Properties" seçin
  • General tab'ında:
  • Cost: 100 (varsayılan) - Düşük değer = yüksek öncelik
  • Replicate every: 180 dakika (varsayılan) - Replikasyon sıklığı
  • Change schedule: Replikasyon saatlerini ayarlayın
5 Domain Controller'ı Site'a Taşıma

Neden: Domain Controller'ları doğru site'lara taşıyarak, replikasyon topolojisini optimize edebilirsiniz.

  • Servers klasörünü genişletin: Tüm DC'leri görüntüler
  • DC'ye sağ tıklayın: "Move" seçin
  • Hedef site'ı seçin: Taşınacak site
  • OK tıklayın: DC taşınır
  • Replication durumunu kontrol edin: "Replication" tab'ında

💻 PowerShell ile Site Yapılandırması (Alternatif)

# Sistem izleme script'i
$servers = @("DC-01", "DC-02", "FileServer-01")
$threshold = 80

foreach ($server in $servers) {
    $cpu = Get-Counter "\\$server\Processor(_Total)\% Processor Time" -SampleInterval 1 -MaxSamples 1
    $memory = Get-Counter "\\$server\Memory\Available MBytes" -SampleInterval 1 -MaxSamples 1
    
    if ($cpu.CounterSamples[0].CookedValue -gt $threshold) {
        Send-MailMessage -To "admin@company.com" -From "monitor@company.com" -Subject "CPU Alert: $server" -Body "CPU usage: $($cpu.CounterSamples[0].CookedValue)%"
    }
    
    if ($memory.CounterSamples[0].CookedValue -lt 1000) {
        Send-MailMessage -To "admin@company.com" -From "monitor@company.com" -Subject "Memory Alert: $server" -Body "Available memory: $($memory.CounterSamples[0].CookedValue) MB"
    }
}

🖥️ Event Viewer Kullanımı

1 Event Viewer Açma

Neden: Event Viewer, sistem olaylarını kategorilere ayırarak gösterir. Her kategori farklı türde olayları içerir.

  • Start → "eventvwr.msc" yazın: Event Viewer'ı doğrudan açar. Bu, en hızlı yöntemdir.
  • Veya Server Manager → Tools → "Event Viewer": Alternatif olarak, Server Manager'dan da açabilirsiniz.
  • Sol panelde "Windows Logs" genişletin: Windows Logs, sistem olaylarını içerir. Application, Security, Setup, System ve Forwarded Events olmak üzere 5 ana kategori vardır.
2 Log Kategorileri

Neden: Her log kategorisi farklı türde olayları içerir. Sorunun türüne göre doğru kategoriyi kontrol etmeniz gerekir.

  • Application: Uygulamalar tarafından oluşturulan olaylar. Örnek: SQL Server hataları, antivirus uyarıları

  • Setup: Sistem kurulum ve yapılandırma olayları. Örnek: Windows Update, rol yüklemeleri
  • System: Windows ve donanım olayları. Örnek: Servis durumları, donanım hataları
  • Forwarded Events: Diğer bilgisayarlardan gelen olaylar (Event Subscriptions kullanılıyorsa)
3 Olay Seviyeleri

Neden: Her olayın bir seviyesi vardır. Bu seviye, olayın önem derecesini gösterir.

  • Critical (Kritik): Sistem çökmesine neden olabilecek ciddi hatalar. Hemen müdahale edilmelidir.
  • Error (Hata): İşlem başarısız oldu. Uygulama çalışmaya devam edebilir ama sorun vardır.
  • Warning (Uyarı): Potansiyel sorun. Şu an çalışıyor ama ileride problem çıkarabilir.
  • Information (Bilgi): Normal işlem. Sorun yok, sadece bilgilendirme.
  • Success Audit (Başarılı Denetim): Güvenlik olayı başarılı. Örnek: Başarılı giriş
  • Failure Audit (Başarısız Denetim): Güvenlik olayı başarısız. Örnek: Başarısız giriş

Performance Monitor ile Sistem İzleme

Neden: Performance Monitor, sistemin gerçek zamanlı performansını izlemenizi sağlar. CPU, RAM, disk ve ağ kullanımını takip edebilirsiniz.

🖥️ Performance Monitor Kullanımı

1 Performance Monitor Açma

Neden: Performance Monitor, sistem kaynaklarının kullanımını grafik olarak gösterir. Bu sayede sistem darboğazlarını tespit edebilirsiniz.

  • Start → "perfmon.msc" yazın: Performance Monitor'ı doğrudan açar.
  • Veya Server Manager → Tools → "Performance Monitor": Alternatif yöntem.
  • Sol panelde "Performance Monitor" tıklayın: Gerçek zamanlı performans grafiklerini gösterir.
2 Önemli Performans Sayaçları

Neden: Her performans sayacı, sistemin farklı bir yönünü ölçer. Bu sayaçları takip ederek sistem sağlığını değerlendirebilirsiniz.

  • CPU Usage (% Processor Time): İşlemci kullanım yüzdesi. %80'in üzerinde sürekli kalıyorsa sorun var.
  • Memory Usage (Available MBytes): Kullanılabilir RAM miktarı. 100MB'ın altına düşerse sorun var.
  • Disk Usage (% Disk Time): Disk kullanım yüzdesi. %90'ın üzerinde sürekli kalıyorsa disk darboğazı var.
  • Network Usage (Bytes Total/sec): Ağ trafiği. Anormal yüksek değerler ağ sorununu işaret eder.

🌐 Ağ Bağlantı Sorunları

1 Ping Testi

  • Localhost testi: ping 127.0.0.1 - TCP/IP yığınının çalışıp çalışmadığını test eder
  • Gateway testi: ping 192.168.1.1 - Yerel ağ bağlantısını test eder
  • DNS testi: ping 8.8.8.8 - İnternet bağlantısını test eder
  • Domain testi: ping google.com - DNS çözümlemesini test eder
2 Tracert ile Rota Takibi

Neden: Tracert, paketlerin hangi yolu izlediğini gösterir. Ağ darboğazlarını ve sorunlu router'ları tespit eder.

  • Rota takibi: tracert google.com - Paketlerin izlediği yolu gösterir
  • Timeout analizi: * işareti, o router'ın yanıt vermediğini gösterir

1 Windows Firewall Yapılandırması

🛡️ Group Policy ile Güvenlik

1 Password Policy

Neden: Güçlü şifre politikaları, hesap güvenliğini artırır. Zayıf şifreler, güvenlik açığı oluşturur.

  • Group Policy Management açın: Start → "gpmc.msc" yazın
  • Default Domain Policy'yi düzenleyin: Domain genelinde geçerli olur
  • Computer Configuration → Policies → Windows Settings → Security Settings → Account Policies → Password Policy:
  • Minimum password length: 8 karakter
  • Password complexity: Enabled
  • Password history: 12 şifre
  • Maximum password age: 90 gün
2 Account Lockout Policy

Neden: Hesap kilitleme politikası, brute force saldırılarını önler. Çok fazla başarısız giriş denemesinde hesabı kilitler.

  • Account Lockout Policy'yi düzenleyin: Default Domain Policy'de
  • Account lockout threshold: 5 deneme
  • Account lockout duration: 30 dakika
  • Reset account lockout counter: 30 dakika

🖥️ RDP Kurulumu ve Yapılandırması

1 RDP Özelliğini Etkinleştirme

  • "Require user authentication for remote connections by using Network Level Authentication" → Enabled: NLA zorunlu kılar

PowerShell Remoting

Neden: PowerShell Remoting, uzaktan PowerShell komutları çalıştırmanızı sağlar. Toplu yönetim ve otomasyon için idealdir.

💻 PowerShell Remoting Kurulumu

1 WinRM Servisini Etkinleştirme

Neden: WinRM (Windows Remote Management), PowerShell Remoting'in temel servisidir. Bu servis olmadan uzaktan PowerShell çalıştıramazsınız.

  • PowerShell'i Administrator olarak açın: Yönetici yetkisi gerekir
  • Enable-PSRemoting komutunu çalıştırın: PowerShell Remoting'i etkinleştirir
  • Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*" -Force: Tüm sunuculara güven verir (test ortamı için)
  • Restart-Service WinRM: WinRM servisini yeniden başlatır
2 Uzaktan PowerShell Bağlantısı

Neden: Uzaktan PowerShell ile birden fazla sunucuyu aynı anda yönetebilirsiniz. Bu, büyük ortamlarda çok faydalıdır.

# Tek sunucuya bağlanma
Enter-PSSession -ComputerName "SERVER01" -Credential (Get-Credential)

# Birden fazla sunucuya komut gönderme
Invoke-Command -ComputerName "SERVER01", "SERVER02" -ScriptBlock { Get-Service }

# Uzaktan dosya kopyalama
Copy-Item -Path "C:\Scripts\*" -Destination "\\SERVER01\C$\Scripts\" -Recurse

# Uzaktan script çalıştırma
Invoke-Command -ComputerName "SERVER01" -FilePath "C:\Scripts\backup.ps1"

📊 Wireshark ile Ağ Analizi

1 Wireshark Kurulumu

Neden: Wireshark, ağ trafiğini gerçek zamanlı olarak yakalar ve analiz eder. Ağ sorunlarını tespit etmek için vazgeçilmezdir.

  • Wireshark'ı indirin: https://www.wireshark.org/
  • Kurulumu yapın: Varsayılan ayarlarla kurun
  • WinPcap veya Npcap yükleyin: Ağ kartı erişimi için gerekli
  • Wireshark'ı Administrator olarak çalıştırın: Ağ kartına erişim için gerekli
2 Ağ Trafiği Yakalama ve Analiz

Neden: Ağ trafiğini analiz ederek, yavaşlık nedenlerini, güvenlik tehditlerini ve protokol sorunlarını tespit edebilirsiniz.

  • Ağ arayüzünü seçin: Ethernet veya Wi-Fi adaptörü
  • "Start capturing" tıklayın: Trafik yakalamaya başlar
  • Filtre uygulayın: Örnek: "ip.addr == 192.168.1.10" (belirli IP'yi filtreler)

🔍 Network Discovery Araçları

1 Nmap ile Ağ Tarama

Neden: Nmap, ağdaki aktif cihazları, açık portları ve çalışan servisleri tespit eder. Güvenlik taraması için kullanılır.

# Ağdaki tüm cihazları tarama
nmap -sn 192.168.1.0/24

# Belirli portları tarama
nmap -p 80,443,3389 192.168.1.10

# Servis versiyonlarını tespit etme
nmap -sV 192.168.1.10

# OS tespit etme
nmap -O 192.168.1.10
2 Advanced IP Scanner

Neden: Advanced IP Scanner, Windows tabanlı kullanıcı dostu bir ağ tarama aracıdır. Grafik arayüzü ile kolay kullanım sağlar.

  • Advanced IP Scanner'ı indirin: Ücretsiz ve portable
  • IP aralığını girin: Örnek: 192.168.1.1-192.168.1.254
  • "Scan" tıklayın: Taramayı başlatır
  • Sonuçları analiz edin: Açık portlar, MAC adresleri, üretici bilgileri
  • RDP/SSH bağlantısı kurun: Doğrudan araçtan bağlanabilirsiniz
