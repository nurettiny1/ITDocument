🔥 FortiGate Firewall

🎯 FortiGate Neden Gerekli?

FortiGate, modern ağ güvenliğinin vazgeçilmez parçasıdır. Neden FortiGate kullanmalıyız?

🔍 FortiGate'in Faydaları:

  • Gelişmiş Güvenlik: Sadece port değil, uygulama seviyesinde koruma
  • Yüksek Performans: Gbps seviyesinde trafik işleme
  • Merkezi Yönetim: Tüm firewall'ları tek yerden yönetim
  • Maliyet Tasarrufu: Tek cihazda birden fazla güvenlik fonksiyonu
  • Otomasyon: Tehdit tespiti ve engelleme otomatik

🏢 Gerçek Hayat Senaryosu: Ransomware Saldırısı

Durum: 200 çalışanlı şirkette ransomware saldırısı

Saldırı: E-posta eki ile gelen zararlı yazılım

Sonuç: Tüm dosyalar şifrelendi, iş durdu

Çözüm: FortiGate ile proaktif koruma ve hızlı müdahale

🚀 FortiGate Özellikleri - Detaylı Açıklama

🛡️ Güvenlik Özellikleri

Stateful Packet Inspection (SPI)

Ne yapar: Paketleri bağlantı durumuna göre analiz eder

Faydası: Sadece meşru trafiğe izin verir

Intrusion Prevention System (IPS)

Ne yapar: Zararlı trafiği gerçek zamanlı tespit eder

Faydası: Saldırıları engeller, güvenlik açıklarını kapatır

Application Control

Ne yapar: Uygulamaları kontrol eder

Faydası: Facebook, YouTube gibi uygulamaları engelleyebilir

🌐 Ağ Özellikleri

VPN (IPSec, SSL-VPN)

Ne yapar: Güvenli uzaktan erişim sağlar

Faydası: Evden ofise güvenli bağlantı

Load Balancing

Ne yapar: Trafiği birden fazla sunucuya dağıtır

Faydası: Performans artışı, yedeklilik

Traffic Shaping

Ne yapar: Bant genişliğini yönetir

Faydası: Kritik uygulamalara öncelik verir

📊 Yönetim Özellikleri

FortiManager

Ne yapar: Birden fazla FortiGate'i yönetir

Faydası: Merkezi yönetim, toplu güncellemeler

FortiAnalyzer

Ne yapar: Logları analiz eder

Faydası: Güvenlik olaylarını raporlar

REST API

Ne yapar: Programatik yönetim sağlar

Faydası: Otomasyon, entegrasyon

⚙️ İlk Yapılandırma - Adım Adım Rehber

Kurulum Adımları

1. 🔌 Fiziksel Bağlantı

Neden önemli: Doğru bağlantı olmadan firewall çalışmaz

  • WAN Port: Internet bağlantısı (ISP'den gelen kablo)
  • LAN Port: İç ağ bağlantısı (switch'e giden kablo)
  • Management Port: Yönetim bilgisayarı (opsiyonel)
  • Güç Kablosu: Güç adaptörü bağlantısı

2. 🌐 IP Yapılandırması

Neden önemli: Firewall'a erişim için IP adresi gerekli

# Varsayılan IP: 192.168.1.99
# Web arayüzüne erişim: https://192.168.1.99
# Varsayılan kullanıcı: admin (şifre yok)

# IP adresini değiştirme
config system interface
    edit "wan1"
        set ip 203.0.113.1 255.255.255.0
        set allowaccess ping https ssh
    next
end

3. 🔐 Güvenlik Ayarları

Neden önemli: Varsayılan şifre güvenlik riski oluşturur

# Admin şifresini değiştir
config system admin
    edit "admin"
        set password "NewPassword123!"
    next
end

# HTTPS erişimini etkinleştir
config system global
    set admin-https-ssl-versions tlsv1-2
    set admin-https-ssl-ciphers ECDHE-RSA-AES128-GCM-SHA256
end

📋 Firewall Politikaları - Gerçek Hayat Senaryoları

Politika Oluşturma - Neden Önemli?

Firewall politikaları neden gerekli?

  • Güvenlik: Zararlı trafiği engeller
  • Kontrol: Hangi trafiğin geçeceğini belirler
  • Performans: Gereksiz trafiği filtreler
  • Uyumluluk: Şirket politikalarını uygular

🏢 Senaryo 1: Küçük Ofis İçin Temel Politikalar

# Temel firewall politikası
config firewall policy
    edit 1
        set name "LAN to WAN"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set logtraffic all
    next
end

# Web trafiğine izin ver
config firewall policy
    edit 2
        set name "Allow HTTP/HTTPS"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "HTTP" "HTTPS"
        set logtraffic all
    next
end

# RDP trafiğini engelle
config firewall policy
    edit 3
        set name "Block RDP"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action deny
        set schedule "always"
        set service "RDP"
        set logtraffic all
    next
end

🔐 VPN Yapılandırması

IPSec VPN

# IPSec VPN Phase 1
config vpn ipsec phase1-interface
    edit "VPN-to-Headquarters"
        set interface "wan1"
        set remote-gw 203.0.113.2
        set psksecret "SharedSecret123!"
        set proposal aes128-sha256
        set dhgrp 14
    next
end

# IPSec VPN Phase 2
config vpn ipsec phase2-interface
    edit "VPN-to-Headquarters"
        set phase1name "VPN-to-Headquarters"
        set proposal aes128-sha256
        set dhgrp 14
    next
end

# VPN politikası
config firewall policy
    edit 10
        set name "VPN Traffic"
        set srcintf "lan"
        set dstintf "VPN-to-Headquarters"
        set srcaddr "192.168.1.0/24"
        set dstaddr "10.0.0.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

SSL-VPN

# SSL-VPN portal oluşturma
config vpn ssl web-portal
    edit "Remote-Access"
        set web-mode enable
        set ip-pools "SSLVPN_POOL"
        set split-tunneling enable
        set split-tunneling-routing-address "192.168.1.0/24"
    next
end

# SSL-VPN kullanıcı grubu
config user group
    edit "SSL-VPN-Users"
        set member "remote-user1" "remote-user2"
    next
end

# SSL-VPN politikası
config firewall policy
    edit 11
        set name "SSL-VPN Access"
        set srcintf "ssl.root"
        set dstintf "lan"
        set srcaddr "SSLVPN_POOL"
        set dstaddr "192.168.1.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

🛡️ Güvenlik Özellikleri

Intrusion Prevention System (IPS)

# IPS profil oluşturma
config ips profile
    edit "High-Security"
        config signature
            config main-class
                edit "all"
                    set action block
                next
            end
        end
    next
end

# IPS politikası
config firewall policy
    edit 20
        set name "IPS Protection"
        set srcintf "wan1"
        set dstintf "lan"
        set srcaddr "all"
        set dstaddr "192.168.1.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
        set ips-sensor "High-Security"
    next
end

Web Filtering

# Web filter profil oluşturma
config webfilter profile
    edit "Corporate-Filter"
        config ftgd-wf
            config filters
                edit 1
                    set category 26
                    set action block
                next
                edit 2
                    set category 57
                    set action block
                next
            end
        end
    next
end

# Web filter politikası
config firewall policy
    edit 21
        set name "Web Filtering"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "192.168.1.0/24"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "HTTP" "HTTPS"
        set webfilter-profile "Corporate-Filter"
    next
end

🔍 Sorun Giderme - Pratik Çözümler

🚨 Yaygın Sorunlar ve Çözümleri

Sorun: VPN Bağlantısı Kurulamıyor

Neden olur: Yanlış yapılandırma, sertifika sorunu

Çözüm adımları:

  1. VPN yapılandırmasını kontrol et
  2. Sertifika süresini kontrol et
  3. Firewall politikalarını kontrol et
  4. Log dosyalarını incele

Sorun: Internet Bağlantısı Yok

Neden olur: Yanlış IP yapılandırması, DNS sorunu

Çözüm adımları:

  1. WAN interface IP'sini kontrol et
  2. DNS sunucularını kontrol et
  3. Gateway ayarlarını kontrol et
  4. ISP bağlantısını test et

Sorun: Web Sitesi Erişilemiyor

Neden olur: Web filtering, firewall politikası

Çözüm adımları:

  1. Web filtering ayarlarını kontrol et
  2. Firewall politikalarını kontrol et
  3. URL kategorilerini kontrol et
  4. Exception listesini kontrol et
