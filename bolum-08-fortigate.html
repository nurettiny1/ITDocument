🔥 FortiGate Firewall

🎯 FortiGate Neden Gerekli?

FortiGate, modern ağ güvenliğinin vazgeçilmez parçasıdır. Neden FortiGate kullanmalıyız?

🔍 FortiGate'in Faydaları:

  • Gelişmiş Güvenlik: Sadece port değil, uygulama seviyesinde koruma
  • Yüksek Performans: Gbps seviyesinde trafik işleme
  • Merkezi Yönetim: Tüm firewall'ları tek yerden yönetim
  • Maliyet Tasarrufu: Tek cihazda birden fazla güvenlik fonksiyonu
  • Otomasyon: Tehdit tespiti ve engelleme otomatik

🏢 Gerçek Hayat Senaryosu:

Problem: 500 çalışanlı şirkette geleneksel firewall yetersiz

Sonuç: Zararlı yazılım saldırıları, veri sızıntıları

Çözüm: FortiGate NGFW ile kapsamlı koruma

🚀 FortiGate Özellikleri

🛡️ Güvenlik Özellikleri

  • Stateful Packet Inspection (SPI)
  • Intrusion Prevention System (IPS)
  • Application Control
  • Web Filtering
  • Antivirus/Antimalware
  • Sandboxing

🌐 Ağ Özellikleri

  • VPN (IPSec, SSL-VPN)
  • Load Balancing
  • Traffic Shaping
  • SD-WAN
  • Wireless Controller
  • Switch Management

📊 Yönetim Özellikleri

  • FortiManager
  • FortiAnalyzer
  • FortiCloud
  • REST API
  • CLI/Web GUI
  • Centralized Logging

🖥️ FortiGate Modelleri

🏠 SOHO Serisi

Küçük ofis ve ev ofisi için

  • FortiGate 30E: 10 kullanıcıya kadar
  • FortiGate 50E: 20 kullanıcıya kadar
  • FortiGate 60E: 50 kullanıcıya kadar

🏢 SMB Serisi

Küçük ve orta ölçekli işletmeler için

  • FortiGate 100E: 100 kullanıcıya kadar
  • FortiGate 200E: 200 kullanıcıya kadar
  • FortiGate 500E: 500 kullanıcıya kadar

🏭 Enterprise Serisi

Büyük işletmeler ve veri merkezleri için

  • FortiGate 1000E: 1000+ kullanıcı
  • FortiGate 3000E: Yüksek performans
  • FortiGate 5000E: Veri merkezi

⚙️ İlk Yapılandırma

Kurulum Adımları

1. 🔌 Fiziksel Bağlantı

  • WAN portuna internet bağlantısı
  • LAN portuna switch bağlantısı
  • Management portuna yönetim bilgisayarı
  • Güç kablosu bağlantısı

2. 🌐 IP Yapılandırması

# Varsayılan IP: 192.168.1.99
# Web arayüzüne erişim: https://192.168.1.99
# Varsayılan kullanıcı: admin (şifre yok)

# IP adresini değiştirme
config system interface
    edit "wan1"
        set ip 203.0.113.1 255.255.255.0
        set allowaccess ping https ssh
    next
end

3. 🔐 Güvenlik Ayarları

# Admin şifresini değiştir
config system admin
    edit "admin"
        set password "NewPassword123!"
    next
end

# HTTPS erişimini etkinleştir
config system global
    set admin-https-ssl-versions tlsv1-2
    set admin-https-ssl-ciphers ECDHE-RSA-AES128-GCM-SHA256
end

📋 Firewall Politikaları

Politika Oluşturma

# Temel firewall politikası
config firewall policy
    edit 1
        set name "LAN to WAN"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set logtraffic all
    next
end

# Web trafiğine izin ver
config firewall policy
    edit 2
        set name "Allow HTTP/HTTPS"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "HTTP" "HTTPS"
        set logtraffic all
    next
end

# RDP trafiğini engelle
config firewall policy
    edit 3
        set name "Block RDP"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action deny
        set schedule "always"
        set service "RDP"
        set logtraffic all
    next
end

🔐 VPN Yapılandırması

IPSec VPN

# IPSec VPN Phase 1
config vpn ipsec phase1-interface
    edit "VPN-to-Headquarters"
        set interface "wan1"
        set remote-gw 203.0.113.2
        set psksecret "SharedSecret123!"
        set proposal aes128-sha256
        set dhgrp 14
    next
end

# IPSec VPN Phase 2
config vpn ipsec phase2-interface
    edit "VPN-to-Headquarters"
        set phase1name "VPN-to-Headquarters"
        set proposal aes128-sha256
        set dhgrp 14
    next
end

# VPN politikası
config firewall policy
    edit 10
        set name "VPN Traffic"
        set srcintf "lan"
        set dstintf "VPN-to-Headquarters"
        set srcaddr "192.168.1.0/24"
        set dstaddr "10.0.0.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

SSL-VPN

# SSL-VPN portal oluşturma
config vpn ssl web-portal
    edit "Remote-Access"
        set web-mode enable
        set ip-pools "SSLVPN_POOL"
        set split-tunneling enable
        set split-tunneling-routing-address "192.168.1.0/24"
    next
end

# SSL-VPN kullanıcı grubu
config user group
    edit "SSL-VPN-Users"
        set member "remote-user1" "remote-user2"
    next
end

# SSL-VPN politikası
config firewall policy
    edit 11
        set name "SSL-VPN Access"
        set srcintf "ssl.root"
        set dstintf "lan"
        set srcaddr "SSLVPN_POOL"
        set dstaddr "192.168.1.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

🛡️ Güvenlik Özellikleri

Intrusion Prevention System (IPS)

# IPS profil oluşturma
config ips profile
    edit "High-Security"
        config signature
            config main-class
                edit "all"
                    set action block
                next
            end
        end
    next
end

# IPS politikası
config firewall policy
    edit 20
        set name "IPS Protection"
        set srcintf "wan1"
        set dstintf "lan"
        set srcaddr "all"
        set dstaddr "192.168.1.0/24"
        set action accept
        set schedule "always"
        set service "ALL"
        set ips-sensor "High-Security"
    next
end

Web Filtering

# Web filter profil oluşturma
config webfilter profile
    edit "Corporate-Filter"
        config ftgd-wf
            config filters
                edit 1
                    set category 26
                    set action block
                next
                edit 2
                    set category 57
                    set action block
                next
            end
        end
    next
end

# Web filter politikası
config firewall policy
    edit 21
        set name "Web Filtering"
        set srcintf "lan"
        set dstintf "wan1"
        set srcaddr "192.168.1.0/24"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "HTTP" "HTTPS"
        set webfilter-profile "Corporate-Filter"
    next
end

🔍 Sorun Giderme

🚨 Yaygın Sorunlar

Sorun: VPN Bağlantısı Kurulamıyor

  1. IPSec Phase 1 durumunu kontrol et
  2. Pre-shared key'i doğrula
  3. Firewall politikalarını kontrol et
  4. NAT traversal ayarlarını kontrol et

Sorun: İnternet Erişimi Yok

  1. WAN interface durumunu kontrol et
  2. Default route'u kontrol et
  3. DNS ayarlarını kontrol et
  4. Firewall politikalarını kontrol et
