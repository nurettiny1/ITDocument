🛡️ Siber Güvenlik

🎯 Siber Güvenlik Neden Kritik?

Siber güvenlik, modern işletmelerin hayatta kalması için zorunludur. Neden siber güvenlik önemli?

🔍 Siber Güvenliğin Faydaları:

  • Veri Koruması: Hassas bilgileri koruma
  • İş Sürekliliği: Saldırılardan sonra hızla toparlanma
  • Maliyet Tasarrufu: Saldırı maliyeti milyonlarca lira
  • Müşteri Güveni: Güvenli sistem = güvenilir şirket
  • Yasal Uyumluluk: GDPR, KVKK gibi yasal gereklilikler

🏢 Gerçek Hayat Senaryosu:

Problem: 1000 çalışanlı şirkette güvenlik önlemleri yetersiz

Sonuç: Ransomware saldırısı, 3 günlük iş durması, 2 milyon lira kayıp

Çözüm: Kapsamlı siber güvenlik stratejisi

⚠️ Siber Tehdit Türleri - Detaylı Analiz

Gerçek Hayat Tehdit Senaryoları

🏢 Senaryo 1: Ransomware Saldırısı - WannaCry

Problem: 1000 çalışanlı şirkette WannaCry ransomware saldırısı

Etki: 3 günlük iş durması, 2 milyon lira kayıp

Çözüm: Hızlı müdahale ve yedekleme stratejisi

Sonuç: %95 veri kurtarma, %90 sistem geri yükleme

Ransomware Özellikleri:
  • Şifreleme: AES-256 ile dosya şifreleme
  • Yayılma: Ağ üzerinden otomatik yayılma
  • Fidye: Bitcoin ile ödeme talebi
  • Zamanlama: 72 saat içinde ödeme

🏢 Senaryo 2: Phishing Saldırısı - CEO Fraud

Problem: CEO'ya yönelik hedefli phishing saldırısı

Etki: 500.000 lira finansal kayıp

Çözüm: E-posta güvenliği ve kullanıcı eğitimi

Sonuç: %99 phishing tespiti, %95 güvenlik artışı

Phishing Teknikleri:
  • Domain Spoofing: Sahte domain kullanımı
  • Social Engineering: Psikolojik manipülasyon
  • Urgency: Aciliyet yaratma
  • Authority: Otorite kullanma

🏢 Senaryo 3: DDoS Saldırısı - Botnet

Problem: E-ticaret sitesine 100 Gbps DDoS saldırısı

Etki: 6 saatlik hizmet kesintisi, 1 milyon lira kayıp

Çözüm: DDoS koruması ve CDN kullanımı

Sonuç: %99.9 uptime, %95 performans artışı

DDoS Saldırı Türleri:
  • Volume-based: Bant genişliği doldurma
  • Protocol-based: Protokol zayıflıklarını kullanma
  • Application-based: Uygulama katmanı saldırıları
  • Reflection: Yansıtma saldırıları

Tehdit Türleri Detaylı Açıklama

🦠 Malware

Kötü amaçlı yazılımlar

  • Virüs: Kendini kopyalayan zararlı kod
  • Worm: Ağ üzerinden yayılan zararlı yazılım
  • Trojan: Masum görünen zararlı yazılım
  • Ransomware: Fidye yazılımı

🎣 Phishing

Kimlik avcılığı saldırıları

  • E-posta Phishing: Sahte e-posta gönderimi
  • Spear Phishing: Hedefli kimlik avcılığı
  • Whaling: Üst düzey yöneticilere yönelik
  • Vishing: Sesli kimlik avcılığı

🔓 Social Engineering

İnsan faktörünü kullanan saldırılar

  • Pretexting: Sahte kimlik kullanma
  • Baiting: Cazip tekliflerle tuzağa düşürme
  • Tailgating: Fiziksel erişim sağlama
  • Quid Pro Quo: Karşılıklı fayda vaadi

🌐 Network Attacks

Ağ tabanlı saldırılar

  • DDoS: Hizmet dışı bırakma saldırısı
  • Man-in-the-Middle: Ortadaki adam saldırısı
  • DNS Spoofing: DNS sahteciliği
  • ARP Spoofing: ARP sahteciliği

🏗️ Güvenlik Çerçeveleri

NIST Cybersecurity Framework

1. 🔍 Identify (Tanımla)

  • Varlıkları belirle
  • Riskleri değerlendir
  • Güvenlik politikaları oluştur
  • Güvenlik gereksinimlerini tanımla

2. 🛡️ Protect (Koruma)

  • Erişim kontrolü uygula
  • Veri güvenliği sağla
  • Güvenlik eğitimi ver
  • Güvenlik araçları kur

3. 🔍 Detect (Tespit)

  • Sürekli izleme yap
  • Anomali tespiti
  • Güvenlik olaylarını analiz et
  • Tehdit istihbaratı kullan

4. ⚡ Respond (Müdahale)

  • Olay müdahale planı
  • Komunikasyon stratejisi
  • Hasar tespiti
  • Kurtarma işlemleri

5. 🔄 Recover (Kurtarma)

  • Yedekleme ve geri yükleme
  • Hizmet sürekliliği
  • Ders çıkarma
  • Süreç iyileştirme

Gerçek Hayat Güvenlik Araçları Senaryoları

🏢 Senaryo 1: Vulnerability Assessment - Nessus

Problem: 500 sunuculu ağda güvenlik açıkları tespit edilemiyor

Çözüm: Nessus ile kapsamlı vulnerability taraması

Sonuç: %95 güvenlik açığı tespiti, %90 risk azaltma

Nessus Kurulum ve Kullanım:
# 1. Nessus kurulumu
sudo apt-get update
sudo apt-get install nessus

# 2. Nessus servisini başlat
sudo systemctl start nessusd
sudo systemctl enable nessusd

# 3. Web arayüzüne erişim
# https://localhost:8834

# 4. Tarama oluşturma
# - Target: 192.168.1.0/24
# - Policy: Basic Network Scan
# - Schedule: Daily at 2 AM

# 5. Rapor oluşturma
# - Executive Summary
# - Technical Details
# - Remediation Steps

🏢 Senaryo 2: Endpoint Protection - CrowdStrike

Problem: 1000 endpoint'te malware tespiti zor

Çözüm: CrowdStrike Falcon ile EDR

Sonuç: %99 malware tespiti, %95 otomatik müdahale

CrowdStrike Kurulum ve Yapılandırma:
# 1. CrowdStrike agent kurulumu
sudo dpkg -i falcon-sensor_6.42.0-15803_amd64.deb

# 2. Agent yapılandırması
sudo /opt/CrowdStrike/falconctl -s --cid=YOUR_CID

# 3. Agent durumunu kontrol et
sudo /opt/CrowdStrike/falconctl -g --cid

# 4. Log dosyalarını kontrol et
sudo tail -f /var/log/crowdstrike/falcon-sensor.log

# 5. Policy yapılandırması
# - Real-time Protection: Enabled
# - Behavioral Analysis: Enabled
# - Machine Learning: Enabled

🏢 Senaryo 3: Network Security - Wireshark

Problem: Ağ trafiğinde şüpheli aktiviteler tespit edilemiyor

Çözüm: Wireshark ile ağ trafiği analizi

Sonuç: %90 anomali tespiti, %95 güvenlik artışı

Wireshark Kullanım Senaryoları:
# 1. Wireshark kurulumu
sudo apt-get install wireshark
sudo usermod -a -G wireshark $USER

# 2. Ağ trafiği yakalama
# - Interface: eth0
# - Filter: tcp port 80
# - Capture: 1000 packets

# 3. Filtreleme örnekleri
# HTTP trafiği: http
# HTTPS trafiği: ssl
# DNS sorguları: dns
# SSH bağlantıları: ssh

# 4. Anomali tespiti
# - Çok fazla bağlantı: tcp.analysis.flags
# - Şüpheli portlar: tcp.port > 1024
# - Büyük paketler: frame.len > 1500

🏢 Senaryo 4: SIEM - Splunk

Problem: Güvenlik olayları merkezi olarak izlenemiyor

Çözüm: Splunk ile SIEM kurulumu

Sonuç: %95 olay tespiti, %90 otomatik müdahale

Splunk Kurulum ve Yapılandırma:
# 1. Splunk kurulumu
wget -O splunk-9.0.0-linux-x86_64.rpm https://download.splunk.com/products/splunk/releases/9.0.0/linux/splunk-9.0.0-linux-x86_64.rpm
sudo rpm -i splunk-9.0.0-linux-x86_64.rpm

# 2. Splunk servisini başlat
sudo /opt/splunk/bin/splunk start --accept-license

# 3. Web arayüzüne erişim
# https://localhost:8000

# 4. Data source ekleme
# - Windows Event Logs
# - Syslog
# - Apache Logs
# - Firewall Logs

# 5. Alert kurulumu
# - Failed login attempts > 5
# - Unusual network traffic
# - File access anomalies

Olay Müdahale Süreci

1. 🔍 Preparation (Hazırlık)

  • Olay müdahale planı hazırla
  • Ekip oluştur ve eğit
  • Araçları hazırla
  • İletişim protokolleri belirle

2. 🚨 Identification (Tespit)

  • Olayı tespit et
  • Kapsamı belirle
  • Öncelik seviyesini belirle
  • İlgili tarafları bilgilendir

3. 🛡️ Containment (Sınırlama)

  • Hasarı sınırla
  • Etkilenen sistemleri izole et
  • Kanıtları koru
  • Geçici çözümler uygula

4. 🔍 Eradication (Yok Etme)

  • Tehdidi tamamen kaldır
  • Zararlı yazılımları temizle
  • Güvenlik açıklarını kapat
  • Sistemleri güncelle

5. 🔄 Recovery (Kurtarma)

  • Sistemleri normale döndür
  • Hizmetleri yeniden başlat
  • İzleme artır
  • Kullanıcıları bilgilendir

6. 📚 Lessons Learned (Dersler)

  • Olayı analiz et
  • Zayıflıkları belirle
  • Süreçleri iyileştir
  • Eğitim ihtiyaçlarını belirle
