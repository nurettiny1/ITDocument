🧩 Group Policy (GPO)

🎯 GPO Neden Kritik?

GPO, domain ortamında kullanıcı ve bilgisayarların merkezi olarak yapılandırılmasını sağlar. Neden GPO kullanıyoruz?

  • Güvenlik Sertleştirme: Parola, lockout, firewall, Defender, BitLocker
  • Standartlaşma: Tüm cihazlarda tutarlı ayarlar
  • Operasyonel Verimlilik: Otomatik dağıtım ve anlık politika değişiklikleri
  • Uyumluluk: CIS/Microsoft benchmark’larına uygunluk

📚 Policy Index (Win10/11, Server 2019/2022)

Aşağıdaki liste, Administrative Templates (Computer/User), Windows Settings/Security Settings ve Preferences kapsamındaki ayarların gezilebilir bir özetidir. Başlıkları açarak detayları görebilirsiniz. Sayfa içi arama kutusunu (Ctrl+K) kullanabilirsiniz.

Computer Configuration User Configuration Security Settings Preferences
Computer Configuration
Policies → Administrative Templates → System
Group Policy
  • Turn off background refresh of Group Policy — Ne yapar: Oturum açıkken periyodik arka plan yenilemeyi kapatır. Etki: Ağ yükünü azaltır; değişikliklerin uygulanması gecikebilir. Tipik kullanım: Zayıf bağlantılı uzak şubeler. Dikkat: Güvenlik güncellemeleri geç uygulanabilir.
  • Configure Group Policy refresh interval for computers — Ne yapar: Arka plan yenileme sıklığını (dakika) ve rastgele gecikmeyi belirler. Tipik: 90+0–30 dk. Dikkat: Sık değerler DC yükünü artırır.
  • Always wait for the network at computer startup and logon — Ne yapar: Senkron GPO işlenmesini zorlar. Tipik: Drive/printer mapping, roaming profile, skript bağımlılıkları. Dikkat: Boot/logon süresi artar.
  • Turn off processing of Legacy run list — Ne yapar: Eski Run girişlerini yoksayar. Güvenlik katılaştırma.
  • Verbose vs normal status messages — Ne yapar: Açılışta ayrıntılı iletiler. Sorun giderme kolaylaşır.
Logon
  • Always use classic logon — Klasik logon ekranını zorlar. Güvenlik/uyumluluk amaçlı.
  • Hide entry points for Fast User Switching — Hızlı kullanıcı değişimini gizler. Tek oturum kurumsal kiosk.
  • Run these programs at user logon — Oturum açılışında çalışacak program listesi. Dikkat: GPP/Scheduled Tasks alternatifleri düşünülmeli.
Power Management
  • Select an Active Power Plan — Etkin güç planını atar (Balanced/High performance). Tipik: Dizüstülerde Balanced.
  • Sleep/Display timeouts — Ekran/uyku zamanlayıcılarını merkezi yönetir. Enerji tasarrufu.
User Profiles
  • Set roaming profile path for all users logging onto this computer — Roaming profil yolu atar.
  • Delete cached copies of roaming profiles — Güvenlik için yerel cache’i temizler.
Policies → Administrative Templates → Network
  • Hardened UNC Paths — SMB erişimleri için güvenlik kısıtları (Mutual Auth, Integrity). Öneri: SYSVOL/NETLOGON’da etkinleştirin.
  • Offline Files — Önbelleğe alma davranışı. Dikkat: Veri tutarlılığı ve çakışmalar.
  • DNS Client — Suffix ekleme, kayıt politikaları.
Policies → Administrative Templates → Windows Components
Windows Update
  • Configure Automatic Updates — Otomatik güncelleme modu/planı. Tipik: 4 (Auto download, schedule install).
  • Specify intranet Microsoft update service location — WSUS adresi. Etki: Trafik ve onaylı yamalar.
  • No auto-restart with logged on users — Kullanıcı oturumunda otomatik yeniden başlatmayı engeller.
Microsoft Defender Antivirus
  • Turn off real-time protection — DEVRE DIŞI bırakmayın (sadece özel senaryoda). Alternatif: ASR ve istisnalar.
  • Exclusions — Yol/süreç/uzantı istisnaları. Dikkat: Saldırı yüzeyini artırır.
  • Cloud-delivered protection — Bulut tabanlı koruma, örnek gönderimi.
BitLocker Drive Encryption
  • Require additional authentication at startup — TPM+PIN vs. Zorunlu ek kimlik doğrulama.
  • Choose how BitLocker-protected OS drives can be recovered — Recovery key AD DS’ye kaydetme.
Remote Desktop Services
  • Require user authentication for remote connections by using NLA — NLA zorunlu. Güvenlik tavsiyesi.
  • Set client connection encryption level — Şifreleme seviyesi.
User Configuration
Policies → Administrative Templates → Start Menu and Taskbar
  • Remove Run menu from Start Menu — “Çalıştır”’ı kaldırır.
  • Remove and prevent access to the Shut Down — Kapat seçeneklerini gizler (kiosk).
  • Force Start to be either full screen size or menu size — Başlat davranışı.
Policies → Administrative Templates → Windows Components → File Explorer
  • Hide these specified drives in My Computer — Sürücüleri gizler.
  • Prevent access to drives from My Computer — Erişimi engeller.
  • Turn off Windows+X hotkeys — Kısayol engelleme.
Windows Settings → Security Settings
  • Account Policies (Password, Lockout, Kerberos) — ayrıntılar yukarıda.
  • Local Policies (Audit/Advanced Audit, User Rights, Security Options)
  • Event Log, Restricted Groups, System Services, Registry, File System
  • Windows Defender Firewall with Advanced Security, IP Security Policies
  • Public Key Policies (Certificates, EFS, Autoenrollment)
  • Software Restriction Policies / AppLocker
  • Name Resolution Policy
Preferences
  • Windows Settings: Drive Maps, Printers, Registry, Files, Shortcuts, Environment, Services, Scheduled Tasks, Network Shares
  • Control Panel Settings: Internet Settings, Regional Options, Power Options, Data Sources, Folder Options
  • Item-level Targeting: Group/OU/Site/IP/OS/Registry vb.

🧩 Administrative Templates – Detaylar

Aşağıdaki başlıklar, Windows 10/11 ve Windows Server 2019/2022 için yerleşik ADMX kapsamındaki yaygın ve kritik policy’lerin açıklamalarını içerir.

System → Removable Storage Access

  • All Removable Storage classes: Deny all access — Ne yapar: Tüm çıkarılabilir depolamaya tam erişimi engeller. Etki: USB bellek/disk okuma-yazma tamamen engellenir. Tipik kullanım: Veri sızıntısını engelleme, yüksek güvenlikli alanlar. Dikkat: İstisnalar için belirli sınıfları ayrı yönetmeniz gerekir.
  • Removable Disks: Deny read access — Ne yapar: USB disklerde okunmayı engeller. Etki: Veri çıkışı engellenir. Tipik kullanım: Sadece yazılım yükleme/policy aracı için yazma gerekirse alternatif düşünün. Dikkat: İş akışlarını bozabilir.
  • Removable Disks: Deny write access — Ne yapar: USB disklerde yazmayı engeller. Etki: Veri kopyalama/çıkarma engellenir. Tipik kullanım: DLP hafif senaryoları. Dikkat: Bazı kurulum medyaları çalışmayabilir.
  • CD and DVD: Deny read/write — Ne yapar: Optik medyada erişimi sınırlar. Tipik kullanım: Eski ortamlarda veri sızıntısı riski azaltma.
  • Floppy / Tape / WPD (MTP cihazlar) — Ne yapar: Eski/özel aygıt sınıflarında erişim kontrolü. Dikkat: WPD telefon bağlantılarını etkiler.
  • All Removable Storage classes: Allow direct access in remote sessions — Ne yapar: Uzak oturumlarda erişime izin istisnası. Dikkat: RDS güvenlik politikanızla uyumluluğu kontrol edin.

System → Device Installation → Device Installation Restrictions

  • Prevent installation of devices not described by other policy settings — Ne yapar: Açıkça izin verilmeyen donanımların kurulmasını engeller. Etki: Yalnızca allowlist cihazlar yüklenir. Tipik kullanım: Kiosk/çağrı merkezi. Dikkat: Sürücü güncellemeleri için allowlist’i güncel tutun.
  • Allow installation of devices that match any of these device IDs — Ne yapar: Donanım Kimliği (DEV/VID) allowlist. Etki: Hedef donanım kurulabilir. Dikkat: Yanlış ID kapsamlarını (parent/child) gözden kaçırmayın.
  • Prevent installation of devices that match any of these device IDs — Ne yapar: Belirli donanım ID’lerini engeller. Tipik kullanım: USB Wi‑Fi dongle yasaklama. Dikkat: Yan etki olarak benzer cihazlar da bloklanabilir.
  • Prevent installation of devices using drivers that match these device setup classes — Ne yapar: Sınıf GUID bloklama. Etki: Geniş kapsamlı engelleme. Dikkat: Yanlış sınıf, kritik aygıtları etkileyebilir.
  • Allow administrators to override Device Installation Restriction policies — Ne yapar: Yöneticilerin kısıtlamayı aşmasına izin. Dikkat: Güvenlik politikanızla uyumlu olsun.
  • Display a custom message when installation is prevented — Ne yapar: Kullanıcıya politika mesajı. Tipik kullanım: Helpdesk yönlendirmesi.

System → Group Policy

  • Always wait for the network at computer startup and logon — Senkron işleme zorlar; GPP/roaming profile/skript bağımlılıkları için önerilir. Etki: Logon süresi artabilir.
  • Configure Group Policy refresh interval for computers — Arka plan yenileme periyodu ve jitter. Öneri: 90 dk + 30 dk rastgele.
  • Turn off background refresh of Group Policy — Arka plan yenilemeyi kapatır. Zayıf hatlarda ağ yükünü azaltır; değişiklikler geç uygulanır.
  • Enable logging for Group Policy Preferences (ilgili bileşenlerde) — Sorun giderme için log ayrıntısını artırır.

System → Logon

  • Hide entry points for Fast User Switching — Kiosk/tek oturum bilgisayarlarda önerilir.
  • Run these programs at user logon — Oturum açılışında çalışan uygulamalar listesi; GPP/Scheduled Tasks çoğu senaryoda daha iyi.
  • Always use classic logon — Uyumluluk/güvenlik amaçlı klasik deneyim.

System → User Profiles

  • Set roaming profile path — Merkezi profil. Dikkat: Ağ gecikmeleri, profil boyutu.
  • Delete cached copies of roaming profiles — Güvenlik için cache temizliği; mobil kullanıcılar için dikkat.

Power Management

  • Select an active power plan — Balanced/High performance ataması.
  • Specify the system sleep/hibernate timeouts — Enerji tasarrufu, pil ömrü.

Network

  • Hardened UNC Paths — \\*\SYSVOL ve \\*\NETLOGON için Mutual Auth/Integrity. Güvenlik tavsiyesi: Etkinleştirin.
  • Offline Files — Önbelleğe alma davranışı; dosya çakışmalarına dikkat.
  • DNS Client — Sonek ekleme, dinamik kayıt; domain ortamında doğru sonek sırası önemli.

Windows Components → Windows Update

  • Configure Automatic Updates — Mod 2/3/4; tipik: 4 + planlı saat.
  • Specify intranet Microsoft update service location — WSUS adresi (http(s)://wsus:8530).
  • No auto-restart with logged on users — Kullanıcı varken otomatik restart yok.
  • Do not include drivers with Windows Updates — Sürücüleri ayırın (kurumsal sürücü yönetimi için).
  • Manage end user experience (Active hours, restart) — Kullanıcı kesintisini azaltır.

Windows Components → Microsoft Defender Antivirus

  • Cloud-delivered protection — İleri tehdit koruması; önerilir.
  • Real-time protection — Devre dışı bırakmayın; istisnaları dikkatle tanımlayın.
  • Exclusions — Yol/süreç/uzantı istisnaları; minimumda tutun.
  • MAPS/Sample submission — Örnek gönderimi; gizlilik gereksinimlerine göre ayarlayın.

Windows Components → BitLocker Drive Encryption

  • Require additional authentication at startup — TPM+PIN; taşınabilirlerde önerilir.
  • Choose how BitLocker-protected OS/data drives can be recovered — Recovery key AD DS’ye/MBAM’e kaydedin.
  • Configure use of hardware-based encryption — Güvenliğe göre yazılım/Donanım şifreleme tercihi.

Windows Components → Remote Desktop Services

  • Require user authentication for remote connections by using NLA — Zorunlu NLA.
  • Set client connection encryption level — Yüksek şifreleme.
  • Limit number of connections / timeouts — Oturum yönetimi.

Remote Desktop Session Host → Device and Resource Redirection

  • Do not allow client printer redirection — Ne yapar: İstemci yazıcılarının oturuma yönlendirilmesini engeller. Etki: Sunucu güvenliği ve sürücü kaosu azalır. Tipik kullanım: Çok kullanıcılı RDS çiftliği. Dikkat: Uçtan yazdırma ihtiyacı olan kullanıcılar için istisna gerebilir.
  • Do not set default client printer to be default printer in a session — Ne yapar: Oturumda varsayılan yazıcıyı istemci yazıcısı yapmaz. Etki: Sunucu tarafı yazıcı politikasını korur. Tipik kullanım: Merkezi yazdırma.
  • Do not allow drive redirection — Ne yapar: İstemci sürücülerini \tsclient\* olarak yönlendirmeyi engeller. Etki: Veri sızıntısı riski azalır. Dikkat: Dosya aktarımı gerekirse alternatif kanal belirleyin.
  • Do not allow clipboard redirection — Ne yapar: Kopyala-yapıştırı engeller. Etki: Veri dışa aktarımı zorlaşır. Dikkat: Kullanıcı deneyimi etkilenir.
  • Do not allow COM port / LPT port / Plug and Play device redirection — Ne yapar: Yerel aygıt yönlendirmelerini engeller. Etki: Yalnızca sunucu kaynakları kullanılır.
  • Audio and video playback/recording redirection — Ne yapar: Ses/Video yönlendirme davranışını belirler. Tipik kullanım: Çağrı merkezi/VDI optimizasyonu.

Remote Desktop Session Host → Security

  • Require user authentication for remote connections by using NLA — Ne yapar: Oturum öncesi NLA şartı. Etki: Kimlik doğrulama güçlenir. Dikkat: Eski istemciler sorun yaşayabilir.
  • Set client connection encryption level — Ne yapar: Şifreleme seviyesini (Low/Client Compatible/High/FIPS) belirler. Tipik kullanım: High veya FIPS. Dikkat: Performans/uyumluluk etkisi.
  • Always prompt for password upon connection — Ne yapar: Tekrar parola sorar. Etki: Kimlik hırsızlığı riski azalır.
  • Set time limit for active but idle Remote Desktop Services sessions — Ne yapar: Idle oturum zaman sınırı. Etki: Kaynak geri kazanımı. Dikkat: İş kaybını önlemek için kullanıcı bilgilendirmesi.

Remote Desktop Session Host → Connections

  • Limit number of connections — Ne yapar: Eşzamanlı oturum sayısını sınırlar. Tipik kullanım: Kaynak planlama, lisans maliyeti kontrolü.
  • Set rules for remote control of Remote Desktop Services user sessions — Ne yapar: Yardım amaçlı oturum izleme/denetleme. Etki: Helpdesk müdahalesi kolaylaşır. Dikkat: Gizlilik/politika.
  • Keep-Alive connections interval — Ne yapar: TCP keep-alive süreleri. Etki: WAN’da kararlılık.

Remote Desktop Session Host → Licensing

  • Use the specified Remote Desktop license servers — Ne yapar: Lisans sunucularını tanımlar. Etki: CAL dağıtımı doğru çalışır.
  • Set the Remote Desktop licensing mode — Ne yapar: Per User / Per Device modu seçer. Dikkat: Uyumluluk ve maliyet.

Remote Desktop Session Host → Session Time Limits

  • Set time limit for disconnected sessions — Ne yapar: Bağlantısı kopan oturumların kapatılma süresi. Etki: Kaynak geri kazanımı.
  • End session when time limits are reached — Ne yapar: Süre dolunca oturumu sonlandırır. Dikkat: Veri kaybını önlemek için kaydetme uyarıları gerekebilir.

Remote Desktop Session Host → Temporary Folders

  • Do not use temporary folders per session — Ne yapar: Oturum başına temp klasörü kullanmaz. Etki: Disk alanı yönetimi değişir.
  • Delete temporary folders on exit — Ne yapar: Oturum kapanınca temp’i temizler. Etki: Gizlilik ve alan kazanımı.

Remote Desktop Session Host → Printer Redirection

  • Redirect only the default client printer — Ne yapar: Sadece varsayılan yazıcı yönlendirilir. Etki: Sürücü karmaşıklığı azalır.
  • Use Remote Desktop Easy Print printer driver first — Ne yapar: Easy Print’i tercih eder. Etki: Sürücü dağıtımını basitleştirir. Dikkat: Bazı yazıcı özellikleri sınırlı olabilir.

Windows Components → Data Collection and Preview Builds

  • Allow Telemetry — Ne yapar: Tanılama verisi seviyesini belirler (Security/Basic/Enhanced/Full). Etki: Gizlilik ve uyumluluk. Tipik kullanım: Kurumlarda Basic/Security. Dikkat: Bazı özellikler daha yüksek seviyeye ihtiyaç duyabilir.
  • Do not show feedback notifications — Ne yapar: Geri bildirim bildirimlerini kapatır. Etki: Kullanıcı kesintisi azalır.
  • Manage preview builds — Ne yapar: Preview build’lere izin/verme ve geciktirme. Tipik: Kurumsal cihazlarda engelle.
  • Toggle user control over Insider builds — Ne yapar: Kullanıcının Insider ayarlarını değiştirmesini kısıtlar.

Windows Components → Windows PowerShell

  • Turn on Script Execution — Ne yapar: Çalıştırma ilkesini belirler (Restricted/AllSigned/RemoteSigned/Unrestricted). Etki: Güvenlik ve otomasyon dengesi. Tipik kullanım: RemoteSigned. Dikkat: İmzalı betikleri tercih edin.
  • Turn on PowerShell Script Block Logging — Ne yapar: Çalışan script bloklarını loglar. Etki: Adli analiz kolaylaşır. Dikkat: Log hacmi artar.
  • Turn on PowerShell Transcription — Ne yapar: Oturum çıktısını dosyaya yazar. Etki: İzlenebilirlik. Dikkat: Hassas veri maskeleme gerebilir.

Windows Components → Windows Update (İleri Ayarlar)

  • Do not connect to any Windows Update Internet locations — Ne yapar: Yalnızca WSUS kullanımı. Etki: Trafik ve kontrol kurumsalda kalır. Dikkat: Microsoft Store etkilenebilir.
  • Allow non-administrators to receive update notifications — Ne yapar: Kullanıcılara bildirim. Etki: Kullanıcı deneyimi. Dikkat: Yardım masası çağrıları artabilir.
  • No auto-restart with logged on users (scheduled installs) — Ne yapar: Planlı yüklemelerde yeniden başlatmayı baskılar. Etki: Kesinti azalır.
  • Configure auto-restart required notifications for updates — Ne yapar: Yeniden başlatma bildirim davranışı. Etki: Kullanıcı bilgilendirme.

Microsoft Defender Antivirus (İleri)

  • Potentially Unwanted Application (PUA) protection — Ne yapar: PUA tespitlerini etkinleştirir (block/audit). Etki: İstenmeyen yazılımlar engellenir. Dikkat: Yanlış pozitifleri izleyin.
  • Scheduled scan — Ne yapar: Planlı tarama günü/saat/davranışı. Etki: Performans ve güvenlik dengesi.
  • Passive mode — Ne yapar: Defender’ı pasif modda çalıştırır (üçüncü taraf AV ile). Dikkat: Yanlış yapılandırma korumayı düşürebilir.

Network → Network Connections

  • Prohibit installation and configuration of network bridges on your DNS domain network — Ne yapar: Bridge/ICS kurulumunu engeller. Etki: İstemci üzerinde yetkisiz yönlendirme engeli. Tipik kullanım: Kurumsal ağ bütünlüğü.
  • Prohibit use of Internet Connection Sharing on your DNS domain network — Ne yapar: ICS’yi engeller. Etki: Güvenlik ve adresleme bütünlüğü.
  • Disable changing proxy settings — Ne yapar: Proxy ayarlarını kilitler. Etki: Trafik yönlendirmesi politikaya uygun kalır.

User Configuration → System (Ctrl+Alt+Del Options)

  • Remove Task Manager — Ne yapar: Görev Yöneticisi’ni kaldırır. Etki: Kötüye kullanımın önlenmesi. Dikkat: Sorun giderme zorlaşır.
  • Remove Change Password — Ne yapar: Parola değiştirme seçeneğini kaldırır. Tipik: Kiosk/konuk hesapları.
  • Remove Lock Computer — Ne yapar: Bilgisayarı kilitle seçeneğini kaldırır. Dikkat: Güvenlik etkisi.

User Configuration → Desktop / Personalization

  • Desktop Wallpaper — Ne yapar: Duvar kağıdını zorunlu kılar (yol/ölçek). Etki: Kurumsal marka. Dikkat: Ağ yoluna erişim.
  • Prevent changing desktop background / theme — Ne yapar: Kişiselleştirmeyi engeller. Etki: Tutarlılık.
  • Hide and disable all items on the desktop — Ne yapar: Masaüstünü boşaltır. Tipik: Kiosk/lab.

User Configuration → Start Menu and Taskbar (Ekstra)

  • Remove access to the context menus for the taskbar — Ne yapar: Görev çubuğu bağlam menülerini kapatır. Etki: Sabit deneyim.
  • Prevent users from customizing their Start Screen — Ne yapar: Başlat özelleştirmesini engeller. Tipik: VDI/Kiosk.
  • Do not keep a history of recently opened documents — Ne yapar: Son öğeler geçmişini kapatır. Etki: Gizlilik.

User Configuration → Start Menu and Taskbar

  • Remove Run menu from Start Menu — “Çalıştır”’ı kaldır.
  • Remove and prevent access to Shut Down — Kapat seçeneklerini gizle.
  • Force Start to be full screen — Başlat boyutu sabit.

User Configuration → Windows Components → File Explorer

  • Hide these specified drives in My Computer — Seçili sürücüleri gizle.
  • Prevent access to drives from My Computer — Erişimi engelle.
  • Turn off Windows+X hotkeys — Kısayolları kapat.

Control Panel (Computer/User)

  • Prohibit access to Control Panel and PC settings — Denetim Masası/Settings’ı kapatır (kiosk/lab).
  • Hide specified Control Panel items — Belirli applet’leri gizler.
  • Show only specified Control Panel items — Sadece listelenen applet’leri gösterir.

Printers

  • Point and Print Restrictions — İmzalanmamış sürücü yüklemesini engelle; kullanıcıdan onay/elevasyon iste.
  • Package Point and Print - Approved servers — Yalnızca onaylı yazıcı sunucularına izin ver.
  • Turn off Legacy Plug and Play resolution — Eski sürücü davranışlarını kısıtlar.

Windows Components → Windows Installer

  • Always install with elevated privileges — Güvenlik nedeniyle önerilmez; kapalı tutun.
  • Disable Windows Installer — Son kullanıcı MSI kurulumlarını kısıtlar.
  • Prohibit removal of updates — Yamaların kaldırılmasını engeller.

Windows Components → Attachment Manager

  • Do not preserve zone information — İnternetten indirilen dosya “zone” bilgisini tutma (önerilmez).
  • Inclusion list for low file types — Düşük risk dosya türleri istisnası (dikkatle).

Windows Components → Credential User Interface

  • Require trusted path for credential entry — Güvenli kimlik giriş akışını zorlar.
  • Enumerate administrator accounts on elevation — Yükseltmede admin hesaplarını listelemeyi engelle.

Windows Components → Remote Assistance

  • Configure Offer Remote Assistance — IT’nin kullanıcıya yardım teklif etmesini etkinleştir/izinleri tanımla.
  • Solicited Remote Assistance — Kullanıcı isteğiyle yardım; güvenlik kapsamını sınırla.

System → Device Guard / Credential Guard

  • Turn On Virtualization Based Security — VBS, HVCI (Memory Integrity). Donanım desteği gerekir.
  • Credential Guard configuration — LSASS kimlik bilgilerini izole eder; kimlik hırsızlığına karşı koruma.

Windows Components → File Explorer (Gelişmiş)

  • Turn off caching of thumbnails in hidden thumbs.db files — Ağ paylaşımlarında kilitlenmeleri azaltır.
  • Hide extensions for known file types — Güvenlik için kapalı önerilir (kötü amaçlı .exe.txt vb.).
  • Turn off display of recent search entries — Gizlilik.

Windows Components → Delivery Optimization

  • Download Mode — LAN/Internet eş paylaşımlı indirme; WAN trafiğini azaltır.

Windows Components → Microsoft Store

  • Only display the private store within the Microsoft Store — Sadece kurumsal mağazayı göster.
  • Disable all apps from Microsoft Store — Uygulama kurulumlarını engeller.

Windows Components → Windows Error Reporting

System → Windows Time Service

  • Time Providers — NTP/NT5DS sağlayıcılarını yapılandır; domain müşterileri için NT5DS, kenar durumlarda NTP.

Windows Components → Windows Defender Application Control (WDAC)

Windows Components → Windows Sandbox

Windows Components → Windows Ink Workspace

Windows Components → Windows Spotlight

🔒 Security Settings (Computer Configuration > Policies > Windows Settings)

Domain genelinde güvenlik sertleştirme için ana bölüm. Aşağıdaki alt başlıkların her biri belirli bir güvenlik alanını yönetir.

1) Account Policies

  • Password Policy: Minimum/maximum yaş, uzunluk, karmaşıklık. Kurumsal parola standardını uygular.
  • Account Lockout Policy: Yanlış deneme eşiği, kilitlenme süresi, reset zamanı. Brute force’u sınırlar.
  • Kerberos Policy: Ticket ömürleri, yenileme, clock skew. Domain kimlik doğrulama güvenliği.

Not: Domain bazlı uygulanır; genelde Default Domain Policy veya ayrı bir SEC-Password-Lockout GPO’su.

2) Local Policies

  • Audit Policy / Advanced Audit: Oturum açma, policy değişikliği, nesne erişimi gibi olayların loglanması. Tercihen Advanced Audit kullanın.
  • User Rights Assignment: Kimler logon olabilir, hizmet olarak oturum açabilir, yedekleme yapabilir vs. En az yetki prensibi.
  • Security Options: NTLM kısıtlamaları, administrator hesabı durumu, UAC, cihaz kuralları gibi yüzlerce ayar.

3) Event Log

  • Application/Security/System log boyutları, saklama ve overwrite davranışı. Olay geçmişinin korunması.

4) Restricted Groups

  • Belirli yerel grupların üyeliklerini merkezi tanımlama. Örn: Administrators grubunda sadece onaylı gruplar.

5) System Services

  • Servislerin başlangıç türleri (Automatic/Manual/Disabled) ve güvenlik izinleri. Saldırı yüzeyini azaltır.

6) Registry

  • Belirli kayıt defteri anahtarlarına izinler, sahiplik, denetim. Sertleştirme ve uygulama konfigürasyonu.

7) File System

  • Klasör/dosya NTFS izinleri ve denetim (auditing). Hassas klasörlerin korunması.

8) Windows Defender Firewall with Advanced Security

  • Domain/Private/Public profiller, inbound/outbound kurallar, IPsec ayarları. Sunucu/istemci bazlı network güvenliği.

9) Public Key Policies

  • Trusted Root/Intermediate CAs, Autoenrollment, EFS, sertifika dağıtımı. Kurumsal PKI entegrasyonu.

10) Software Restriction Policies / AppLocker

  • SRP: Yol/hash/zone tabanlı temel kısıtlama.
  • AppLocker: Publisher/Path/Hash kuralları ile modern ve ayrıntılı uygulama kontrolü.

11) IP Security Policies

  • IPsec politikaları ile network trafiğini şifreleme/kimlik doğrulama. Güncel senaryolarda genelde Firewall/IPsec ile yönetilir.

12) Advanced Audit Policy Configuration

  • Detaylı denetim kategorileri (Logon, Object Access, Policy Change, DS Access, Account Logon, Privilege Use, System, Detailed Tracking).

13) Name Resolution Policy

  • DNS Client policy’leri (ör. DNSSEC). Kurumsal ad çözümlemede güvenlik.

En İyi Uygulamalar (Security Settings)

  • Parola/Lockout’ı domain seviyesinde tek GPO’da yönetin; test OU’su ile pilot yapın.
  • Audit’i Advanced Audit ile ayrıntılı tanımlayın; SIEM’e iletin.
  • Local admin’leri Restricted Groups (veya LAPS) ile kontrol edin.
  • Firewall ve IPsec kurallarını profillere göre ayrı GPO’larla bölün.
  • AppLocker’ı önce Audit modda deneyin, sonra Enforce’a alın.

🏗️ GPO Temelleri ve İşleyiş

  • LSDOU: Local → Site → Domain → OU sırasıyla uygulanır.
  • Inheritance: Üstten alta devralma; Block Inheritance ile engellenebilir, Enforced (No override) ile zorlanabilir.
  • Link Order: Aynı OU’daki link sırası önemlidir (düşük numara daha yüksek öncelik).
  • User vs Computer: Politikanın hedefi farklıdır; Loopback ile kullanıcı ayarları bilgisayar bağlamında uygulanabilir.

🎯 Scope, Filtreleme ve Loopback

  • Security Filtering: GPO yalnızca belirli grup/kullanıcı/bilgisayarlara uygulatılır.
  • WMI Filtering: OS sürümü, donanım, network gibi koşullara göre hedefleme.
  • Loopback Processing: Kiosk/VDI/Lab senaryolarında kullanıcı ayarlarını bilgisayara göre zorla.
# PowerShell ile GPO kapsamı ve link işlemleri (GroupPolicy modülü)
# Not: Group Policy Management Console (GPMC) yüklü olmalıdır.
# New-GPO, New-GPLink, Set-GPPermission gibi cmdlet'ler kullanılır.

# Örnek: Yeni GPO oluştur ve OU'ya linkle
New-GPO -Name "SEC-Baseline-Workstations" | New-GPLink -Target "OU=Workstations,DC=contoso,DC=local" -LinkEnabled Yes

# Örnek: Security filtering için izin ekle (Read + Apply)
$gpo = Get-GPO -Name "SEC-Baseline-Workstations"
Set-GPPermission -Name $gpo.DisplayName -TargetName "GG-Workstations" -TargetType Group -PermissionLevel GpoApply

# Loopback etkinleştirme (bilgilendirme)
# User Configuration > Policies > Administrative Templates > System > Group Policy >
# "User Group Policy loopback processing mode" = Merge veya Replace

📁 Administrative Templates (ADMX) – Computer ve User

ADMX/ADML şablonları ile binlerce sistem ve kullanıcı ayarı merkezi yönetilir. Central Store kullanarak tüm DC’lerde tutarlılık sağlayın.

Computer Configuration > Administrative Templates – Öne Çıkan Alanlar

  • System: Logon, Profiles, Power Management, Group Policy, Scripts. (Örn: Always wait for the network at computer startup and logon)
  • Network: Offline Files, DNS Client, NLA. (Örn: Hardened UNC Paths)
  • Windows Components: Windows Update, Defender, Remote Desktop Services, BitLocker, SmartScreen.
  • Control Panel: Settings visibility kısıtlamaları.
  • Printers: Point and Print Restrictions, driver yükleme kısıtları.

Tipik Kullanımlar (Computer)

  • Windows Update politikası (Automatic Updates schedule, deferral)
  • Defender ayarları (Real-time protection, ASR kuralları – ADMX’e bağlı)
  • RDP sertleştirme (NLA, şifreleme seviyeleri)
  • BitLocker politikaları (OS/Data drive, TPM/PIN, recovery key saklama)
  • SmartScreen ve Exploit Guard ilkeleri

User Configuration > Administrative Templates – Öne Çıkan Alanlar

  • Start Menu and Taskbar: Menü öğeleri, arama, sabitlenen uygulamalar.
  • Desktop: Simgeler, duvar kağıdı, kısayol kısıtlamaları.
  • Control Panel: Denetim Masası erişim ve alt öğe kısıtlamaları.
  • Windows Components: OneDrive, Edge/IE, File Explorer, Teams/Office ADMX (yüklüyse).
# ADMX Central Store yolu (bilgilendirme)
# \\DOMAIN\SYSVOL\DOMAIN\Policies\PolicyDefinitions
# Yeni ürün ADMX’lerini buraya kopyalayın (örn. Office, Chrome, Edge).

En İyi Uygulamalar (Administrative Templates)

  • ADMX Central Store kullanın; versiyonları dokümante edin.
  • Bilgisayar ve kullanıcı ayarlarını ayrı GPO’larda yönetin.
  • Windows Update/Defender/BitLocker gibi kritik politikaları ayrı isimli GPO’lara bölün.
  • Değişiklikleri pilot OU’da test ettikten sonra genişletin.

🧰 Preferences – Windows ve Control Panel Settings

GPP, politik olmayan (non-enforced) ama güçlü dağıtım kabiliyetleri sunar. Item-level Targeting ile koşullu hedefleme yapabilirsiniz.

Windows Settings

  • Drive Maps: Paylaşımları harflerle bağlama. (Örn: \\fs\departman → H:)
  • Printers: Yazıcı dağıtımı (Default printer, bağlantı).
  • Registry: Kayıt defteri anahtar/değer atama.
  • Files/Shortcuts: Dosya kopyalama, kısayol oluşturma.
  • Environment: Ortam değişkenleri yönetimi.
  • Services: Servis başlangıç türü ve durum ayarı.
  • Scheduled Tasks: Görev zamanlama (PowerShell script tetikleme vb.).
  • Network Shares: Paylaşım oluşturma/ayar.

Control Panel Settings

  • Internet Settings: Proxy, güvenlik bölgeleri.
  • Regional Options: Dil/bölge/klavye.
  • Power Options: Güç planları.
  • Data Sources / Folder Options: Uygulama ve görünüm ayarları.

Item-level Targeting (ILT)

  • Security Group, OU, Site, IP/MAC, OS sürümü, bilgisayar adı, LDAP sorgusu, Registry değeri vb. koşullarla hedefleme.
  • Örnek: “GG-Finance” grubundaki kullanıcılar için \\fs\finance sürücüsünü bağla; lokasyon “IST” ise yazıcı “PRN-IST-01”.
# Öneri (GUI yolu):
# User Configuration > Preferences > Windows Settings > Drive Maps
# New > Mapped Drive > Location: \\fs\finance | Label: Finance | Reconnect: Yes
# Common > Item-level Targeting: Security Group = GG-Finance; Computer Location = IST

En İyi Uygulamalar (Preferences)

  • Script’lerle yaptığınız kopyalama/drive/printer dağıtımlarını GPP’ye taşıyın.
  • ILT ile kapsamı daraltın; karmaşık hedefler için koşulları gruplayın.
  • Bilgisayar ve kullanıcı GPP öğelerini ayrı GPO’larda yönetin.
  • “Replace” yerine “Update” davranışını tercih edin (gereksiz sil-yap yeniden riskini azaltır).

📜 Scripts ve 💽 Software Settings

GPO ile script çalıştırma ve MSI tabanlı yazılım dağıtımı yapılabilir. Büyük ölçekte yazılım dağıtımı için genellikle MEM/Intune, SCCM veya üçüncü taraf MDM tercih edilir; ancak küçük/orta senaryolarda GPO yeterlidir.

Scripts

  • Computer Configuration > Policies > Windows Settings > Scripts (Startup/Shutdown): Makine açılış/kapanışında çalışır.
  • User Configuration > Policies > Windows Settings > Scripts (Logon/Logoff): Kullanıcı oturum açma/kapatmada çalışır.
  • Sıra ve zamanlama: Startup önce, sonra Logon; Shutdown ve Logoff kapanışta. “Run scripts synchronously” kritik durumlarda önerilir.
  • Konum: Scriptleri \\DOMAIN\\SYSVOL\\DOMAIN\\Scripts veya erişimi olan paylaşımlarda tutun.
# Basit bir PowerShell başlatma scripti (örnek)
# File: \\DOMAIN\SYSVOL\DOMAIN\Scripts\Startup\Init-Workstation.ps1
Write-EventLog -LogName Application -Source "GPO-Startup" -EventId 1001 -EntryType Information -Message "Startup policy executed."

# GPO > Computer Configuration > Policies > Windows Settings > Scripts (Startup)
# Add > Powershell Scripts > Script Name: Init-Workstation.ps1 (Parameters boş kalabilir)
# Not: Execution Policy, Administrative Templates ile yönetilebilir (Turn on Script Execution)

En İyi Uygulamalar (Scripts)

  • Script’leri imzalayın veya güvenilir kaynaktan dağıtın; olay kaydı yazdırın.
  • Uzun süren işlemler için zamanlayıcı (Scheduled Task) veya yönetim aracı kullanın.
  • SYSVOL’de gereksiz büyük dosyalardan kaçının; versiyon ve değişim takibi yapın.

Software Settings (Software Installation)

  • Computer/User > Policies > Software Settings > Software Installation: MSI paketlerinin Assigned/Published dağıtımı.
  • Assigned vs Published: Computer Assigned başta yüklenir; User Published kullanıcı Programlar ve Özellikler’den kurabilir.
  • Upgrades/Removal: Mevcut sürümü yükseltme (Upgrades) ve kaldırma seçenekleri (Uninstall).
  • Kaynak yolu: MSI ve MST dosyalarını erişilebilir bir UNC yolunda tutun (\fileserver\packages\vendor\product\version).
  • Transforms (MST): Sessiz kurulum, özellik/özelleştirmeler için MST kullanın.
# Öneri: Paket paylaşımı yapısı
# \\fileserver\packages\Vendor\Product\1.2.3\Product.msi
# \\fileserver\packages\Vendor\Product\1.2.3\Product.mst (isteğe bağlı)

En İyi Uygulamalar (Software Installation)

  • MSI’ları bilgisayar kapsamından (Assigned) dağıtın; oturum açmadan önce kurulur.
  • Kaynaktaki paket yolunu sabit ve yüksek erişilebilirlikte tutun; izinleri doğru ayarlayın.
  • Yükseltmeleri (Upgrades) dikkatle tanımlayın; eski sürümleri temiz kaldırın.
