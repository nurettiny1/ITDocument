🖥️ Windows Server Kurulumu

🎯 Windows Server Neden Gerekli?

Windows Server, modern işletmelerin altyapısının temelidir. Neden Windows Server kullanmalıyız?

🔍 Windows Server'ın Faydaları:

  • Merkezi Yönetim: Tüm sunucuları tek yerden yönetebilirsiniz
  • Güvenlik: Enterprise seviye güvenlik özellikleri
  • Ölçeklenebilirlik: İşletme büyüdükçe sistem büyür
  • Uyumluluk: Microsoft ekosistemi ile tam entegrasyon
  • Maliyet Etkinliği: Lisans maliyeti vs. performans dengesi

🏢 Gerçek Hayat Senaryosu:

Problem: 200 çalışanlı şirkette her departman kendi sunucusunu yönetiyor

Sonuç: Kaos, güvenlik açıkları, maliyet artışı

Çözüm: Windows Server ile merkezi altyapı

🎯 Hangi Sürümü Seçmeliyiz?

🏠 Küçük İşletme (1-25 kullanıcı)

Önerilen: Windows Server Essentials

Neden: Basit yönetim, düşük maliyet, otomatik yedekleme

# Essentials için tipik kullanım
- Dosya paylaşımı
- E-posta sunucusu
- Uzaktan erişim
- Otomatik yedekleme

🏢 Orta İşletme (26-500 kullanıcı)

Önerilen: Windows Server Standard

Neden: VM desteği, AD, maliyet/performans dengesi

# Standard için tipik kullanım
- Active Directory Domain Controller
- Hyper-V sanallaştırma
- IIS web sunucusu
- SQL Server

🏭 Büyük İşletme (500+ kullanıcı)

Önerilen: Windows Server Datacenter

Neden: Sınırsız VM, gelişmiş güvenlik, yüksek kullanılabilirlik

# Datacenter için tipik kullanım
- Çoklu VM ortamı
- Failover clustering
- Software-defined networking
- Gelişmiş güvenlik

🎯 Kurulum Öncesi Hazırlık

Windows Server kurulumu kritik bir süreçtir. Neden dikkatli planlama gerekli?

✅ Kurulum Öncesi Kontrol Listesi

  • Donanım Uyumluluğu: HCL (Hardware Compatibility List) kontrolü
  • Lisans Planlaması: Hangi sürüm, kaç lisans gerekli
  • Ağ Planlaması: IP adresleri, domain adı, DNS
  • Güvenlik Planlaması: Firewall kuralları, güvenlik politikaları
  • Yedekleme Stratejisi: Sistem yedekleme planı

  • Ağ: 10 Gigabit Ethernet
  • Maliyet: ₺80,000 - ₺120,000
# Orta işletme için tipik kullanım

🔧 Adım Adım Kurulum Süreci

1. 🖥️ Kurulum Medyası Hazırlama

Neden önemli? Doğru medya = başarılı kurulum

📥 ISO İndirme
# Microsoft Volume Licensing Service Center
# 1. Microsoft VLSC'ye giriş yap
# 2. Windows Server 2022 seç
# 3. ISO dosyasını indir (4-5 GB)
# 4. SHA256 hash'ini doğrula
💾 USB Medya Oluşturma
# Rufus ile USB oluşturma
# 1. Rufus'u indir ve çalıştır
# 2. USB'yi seç (8GB+)
# 3. ISO dosyasını seç
# 4. GPT + UEFI seç
# 5. "Start" butonuna bas

2. 🔧 BIOS/UEFI Yapılandırması

Neden önemli? Doğru ayarlar = güvenli ve hızlı kurulum

⚙️ BIOS Ayarları
# BIOS/UEFI Ayarları
1. Secure Boot: Enabled
2. UEFI Mode: Enabled
3. Legacy Boot: Disabled
4. TPM 2.0: Enabled
5. Virtualization: Enabled (Intel VT-x/AMD-V)
6. Boot Order: USB First
🔒 Güvenlik Ayarları
# Güvenlik Ayarları
1. Admin Password: Güçlü şifre
2. Boot Password: Etkinleştir
3. TPM: Enabled
4. Secure Boot: Enabled
5. Intel TXT: Enabled (varsa)

3. 📀 Kurulum Başlatma

Neden önemli? Doğru seçimler = sorunsuz kurulum

🚀 Kurulum Adımları
# Kurulum Süreci
1. USB'den boot et
2. "Install Now" seç
3. Product Key gir (varsa)
4. Edition seç (Standard/Datacenter)
5. License terms kabul et
6. "Custom: Install Windows only" seç
💾 Disk Bölümleme
# Disk Bölümleme Stratejisi
# Küçük İşletme
C: 100GB (Sistem)
D: Kalan (Veri)

# Orta İşletme  
C: 200GB (Sistem)
D: 500GB (Veri)
E: 200GB (Logs)

# Büyük İşletme
C: 500GB (Sistem)
D: 1TB (Veri)
E: 500GB (Logs)
F: 1TB (Backup)

🎭 Sunucu Rolleri - Hangi Rolleri Kurmalıyız?

Windows Server'da rolleri seçmek kritik bir karardır. Neden doğru rol seçimi önemli?

🎯 Rol Planlama Stratejisi

🏠 Küçük İşletme Rolleri

Önerilen Roller:

  • Active Directory Domain Services: Kullanıcı yönetimi
  • DNS Server: Domain çözümleme
  • File and Storage Services: Dosya paylaşımı
  • Print and Document Services: Yazıcı yönetimi
# Küçük işletme için temel roller
Install-WindowsFeature -Name AD-Domain-Services, DNS, File-Services, Print-Services -IncludeManagementTools

🏢 Orta İşletme Rolleri

Önerilen Roller:

  • Active Directory Domain Services: Merkezi yönetim
  • DNS Server: Domain çözümleme
  • DHCP Server: IP yönetimi
  • Hyper-V: Sanallaştırma
  • IIS: Web servisleri
# Orta işletme için gelişmiş roller
Install-WindowsFeature -Name AD-Domain-Services, DNS, DHCP, Hyper-V, IIS -IncludeManagementTools

🏭 Büyük İşletme Rolleri

Önerilen Roller:

  • Active Directory Domain Services: Merkezi yönetim
  • DNS Server: Domain çözümleme
  • DHCP Server: IP yönetimi
  • Hyper-V: Sanallaştırma
  • Failover Clustering: Yüksek kullanılabilirlik
  • Windows Server Update Services: Güncelleme yönetimi
# Büyük işletme için enterprise roller
Install-WindowsFeature -Name AD-Domain-Services, DNS, DHCP, Hyper-V, Failover-Clustering, UpdateServices -IncludeManagementTools

DNS Server - Neden Kritik?

Faydaları: Domain çözümleme, hızlı erişim, merkezi yönetim

🖥️ GUI ile Kurulum (Kolay Yöntem)
  1. Server Manager açın
  2. "Add roles and features" tıklayın
  3. "Role-based or feature-based installation" seçin
  4. "DNS Server" rolünü seçin
  5. "Add Features" tıklayın
  6. "Next" ile devam edin
  7. "Install" tıklayın
⚡ PowerShell ile Kurulum (Hızlı Yöntem)
# DNS Server kurulumu
Install-WindowsFeature -Name DNS -IncludeManagementTools

# DNS zone oluşturma
Add-DnsServerPrimaryZone -Name "company.local" -ZoneFile "company.local.dns"

# A kaydı ekleme
Add-DnsServerResourceRecordA -Name "server1" -ZoneName "company.local" -IPv4Address "192.168.1.10"

# CNAME kaydı ekleme
Add-DnsServerResourceRecordCName -Name "www" -ZoneName "company.local" -HostNameAlias "server1.company.local"
🎯 GUI ile DNS Yönetimi
  1. DNS Manager açın (dnsmgmt.msc)
  2. Forward Lookup Zones sağ tık
  3. "New Zone" seçin
  4. Zone name: company.local
  5. "Create a new file" seçin
  6. Dynamic updates: "Allow both"

DHCP Server - Neden Gerekli?

Faydaları: Otomatik IP dağıtımı, merkezi yönetim, hata azaltma

🖥️ GUI ile Kurulum (Kolay Yöntem)
  1. Server Manager açın
  2. "Add roles and features" tıklayın
  3. "Role-based or feature-based installation" seçin
  4. "DHCP Server" rolünü seçin
  5. "Add Features" tıklayın
  6. "Next" ile devam edin
  7. "Install" tıklayın
⚡ PowerShell ile Kurulum (Hızlı Yöntem)
# DHCP Server kurulumu
Install-WindowsFeature -Name DHCP -IncludeManagementTools

# DHCP scope oluşturma
Add-DhcpServerv4Scope -Name "LAN" -StartRange 192.168.1.100 -EndRange 192.168.1.200 -SubnetMask 255.255.255.0

# DHCP seçenekleri
Set-DhcpServerv4OptionValue -OptionId 3 -Value "192.168.1.1" -ScopeId "192.168.1.0"
Set-DhcpServerv4OptionValue -OptionId 6 -Value "192.168.1.10" -ScopeId "192.168.1.0"
🎯 GUI ile DHCP Yönetimi
  1. DHCP Manager açın (dhcpmgmt.msc)
  2. Server sağ tık → "New Scope"
  3. Scope name: LAN
  4. IP range: 192.168.1.100 - 192.168.1.200
  5. Subnet mask: 255.255.255.0
  6. Router (Gateway): 192.168.1.1
  7. DNS Server: 192.168.1.10

👥 Dizin Servisleri - Neden Kritik?

Active Directory olmadan: Kullanıcı kaosu, güvenlik açıkları, merkezi yönetim yok

Active Directory Domain Services - Neden Gerekli?

Faydaları: Merkezi kullanıcı yönetimi, güvenlik, grup politikaları

🖥️ GUI ile Kurulum (Kolay Yöntem)
  1. Server Manager açın
  2. "Add roles and features" tıklayın
  3. "Role-based or feature-based installation" seçin
  4. "Active Directory Domain Services" rolünü seçin
  5. "Add Features" tıklayın
  6. "Next" ile devam edin
  7. "Install" tıklayın
  8. "Promote this server to a domain controller" tıklayın
  9. "Add a new forest" seçin
  10. Root domain name: company.local
  11. Safe Mode Password: Güçlü şifre girin
⚡ PowerShell ile Kurulum (Hızlı Yöntem)
# AD DS rolü ekleme
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools

# Domain Controller kurulumu
Install-ADDSForest -DomainName "company.local" -SafeModeAdministratorPassword (ConvertTo-SecureString "Password123!" -AsPlainText -Force)

# Kullanıcı oluşturma
New-ADUser -SamAccountName "john.doe" -UserPrincipalName "john.doe@company.local" -Name "John Doe" -GivenName "John" -Surname "Doe" -Enabled $true -AccountPassword (ConvertTo-SecureString "TempPass123!" -AsPlainText -Force)

# Grup oluşturma
New-ADGroup -Name "IT Department" -GroupScope Global -GroupCategory Security
🎯 GUI ile AD Yönetimi
  1. Active Directory Users and Computers açın (dsa.msc)
  2. Users klasörüne sağ tık
  3. "New" → "User" seçin
  4. First name: John
  5. Last name: Doe
  6. User logon name: john.doe
  7. Password: Güçlü şifre
  8. "User must change password at next logon" işaretleyin

🔒 Güvenlik Yapılandırması

Temel Güvenlik Ayarları

1. 🛡️ Windows Defender

# Windows Defender'ı etkinleştir
Set-MpPreference -DisableRealtimeMonitoring $false

# Güvenlik güncellemelerini kontrol et
Get-WindowsUpdate -AcceptAll -Install -AutoReboot

2. 🔐 Güvenlik Duvarı

# Windows Firewall'ı etkinleştir
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True

# Özel kural oluştur
New-NetFirewallRule -DisplayName "Custom Rule" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow

3. 🔑 Kullanıcı Hesapları

# Administrator hesabını yeniden adlandır
Rename-LocalUser -Name "Administrator" -NewName "AdminUser"

# Guest hesabını devre dışı bırak
Disable-LocalUser -Name "Guest"

# Şifre politikası uygula
net accounts /minpwlen:8 /maxpwage:90

🔍 Sorun Giderme

🚨 Yaygın Sorunlar

Sorun: Sunucu Açılmıyor

  1. Fiziksel bağlantıları kontrol et
  2. BIOS/UEFI ayarlarını kontrol et
  3. Boot sırasını kontrol et
  4. Hardware testleri yap

Sorun: Yavaş Performans

  1. Task Manager ile CPU/RAM kullanımını kontrol et
  2. Disk performansını analiz et
  3. Gereksiz servisleri kapat
  4. Antivirus taraması yap

🛠️ Faydalı Komutlar

# Sistem bilgilerini görüntüle
Get-ComputerInfo

# Servis durumlarını kontrol et
Get-Service | Where-Object {$_.Status -ne "Running"}

# Disk kullanımını kontrol et
Get-WmiObject -Class Win32_LogicalDisk | Select-Object DeviceID, @{Name="Size(GB)";Expression={[math]::Round($_.Size/1GB,2)}}, @{Name="FreeSpace(GB)";Expression={[math]::Round($_.FreeSpace/1GB,2)}}

# Event log'ları kontrol et
Get-EventLog -LogName System -Newest 10

# Ağ bağlantılarını kontrol et
Get-NetAdapter | Where-Object {$_.Status -eq "Up"}
