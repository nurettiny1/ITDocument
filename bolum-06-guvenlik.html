🔒 AppLocker

AppLocker Nedir?

AppLocker, hangi uygulamaların sistemde çalışabileceğini kontrol eden bir güvenlik özelliğidir. Executable dosyalar, scriptler ve Windows Installer paketlerini yönetir.

AppLocker Kuralları

📱 Executable Rules

EXE ve COM dosyalarını kontrol eder

# PowerShell ile executable kuralı oluşturma
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Program Files\*" -User Everyone

# Belirli uygulamayı engelle
New-AppLockerPolicy -RuleType Path -Action Deny -Path "C:\Temp\malware.exe" -User Everyone

📜 Script Rules

PowerShell, VBScript, JavaScript dosyalarını kontrol eder

# Script kuralı oluşturma
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Scripts\*" -User ITAdmins

# PowerShell scriptlerini kısıtla
New-AppLockerPolicy -RuleType Path -Action Deny -Path "*.ps1" -User RegularUsers

📦 Installer Rules

MSI ve MSP dosyalarını kontrol eder

# Installer kuralı oluşturma
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Software\*" -User ITAdmins

# Sadece imzalı installer'lara izin ver
New-AppLockerPolicy -RuleType Publisher -Action Allow -Publisher "Microsoft Corporation" -User Everyone

AppLocker Yapılandırması

1. AppLocker Servisini Etkinleştir

# AppLocker servisini başlat
Start-Service -Name "AppIDSvc"

# Servisi otomatik başlat
Set-Service -Name "AppIDSvc" -StartupType Automatic

2. GPO ile AppLocker Kuralları Uygula

# AppLocker politikasını GPO'ya uygula
Set-AppLockerPolicy -XmlPolicy (Get-Content "C:\AppLockerPolicy.xml" -Raw)

# Mevcut kuralları görüntüle
Get-AppLockerPolicy -Effective

🔐 BitLocker

BitLocker Özellikleri

  • Disk Şifreleme: Tüm disk içeriğini şifreler
  • TPM Desteği: Trusted Platform Module ile güvenlik
  • Recovery Key: Şifre kurtarma anahtarı
  • Auto-unlock: Otomatik şifre çözme

BitLocker Yapılandırması

# BitLocker'ı etkinleştir
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -UsedSpaceOnly

# Recovery key'i kaydet
$recoveryKey = (Get-BitLockerVolume -MountPoint "C:").KeyProtector | Where-Object {$_.KeyProtectorType -eq "RecoveryPassword"}
$recoveryKey.RecoveryPassword | Out-File "C:\BitLockerRecoveryKey.txt"

# BitLocker durumunu kontrol et
Get-BitLockerVolume

# BitLocker'ı devre dışı bırak
Disable-BitLocker -MountPoint "C:"

🔥 Windows Firewall

Firewall Kuralları

📥 Inbound Rules

Gelen bağlantıları kontrol eder

# HTTP trafiğine izin ver
New-NetFirewallRule -DisplayName "Allow HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow

# RDP'yi sadece belirli IP'lerden kabul et
New-NetFirewallRule -DisplayName "Allow RDP from IT" -Direction Inbound -Protocol TCP -LocalPort 3389 -RemoteAddress "192.168.1.0/24" -Action Allow

# Tüm gelen trafiği engelle
New-NetFirewallRule -DisplayName "Block All Inbound" -Direction Inbound -Action Block

📤 Outbound Rules

Giden bağlantıları kontrol eder

# HTTPS trafiğine izin ver
New-NetFirewallRule -DisplayName "Allow HTTPS" -Direction Outbound -Protocol TCP -RemotePort 443 -Action Allow

# DNS sorgularına izin ver
New-NetFirewallRule -DisplayName "Allow DNS" -Direction Outbound -Protocol UDP -RemotePort 53 -Action Allow

# Sosyal medya sitelerini engelle
New-NetFirewallRule -DisplayName "Block Social Media" -Direction Outbound -RemoteAddress "facebook.com", "twitter.com" -Action Block

# 1. Access control monitoring

Get-ChildItem -Path "C:\PatientData" -Recurse -File |

        }
    }

Get-WBBackupSet | Where-Object {$_.BackupTime -gt (Get-Date).AddDays(-7)} |

🛡️ Zero Trust Security Model

Prensip: "Never trust, always verify" - Hiçbir şeye güvenme, her zaman doğrula

graph TD A[User Request
Access Attempt] --> B[Identity Verification
Multi-Factor Auth] B --> C[Device Trust
Compliance Check] C --> D[Network Trust
Location/Time Analysis] D --> E[Application Trust
Least Privilege Access] E --> F[Data Trust
Encryption/Classification] G[Continuous Monitoring
Behavioral Analysis] --> H[Risk Assessment
Real-time Scoring] I[Policy Engine
Dynamic Policies] --> J[Access Decision
Allow/Deny/Challenge] 
# Zero Trust implementation
# 1. Identity verification
$user = Get-ADUser -Identity $username
$mfaStatus = Get-MFAStatus -User $user
$deviceCompliance = Get-DeviceCompliance -Device $deviceId

# 2. Risk assessment
$riskScore = 0
if (-not $mfaStatus.Enabled) { $riskScore += 30 }
if (-not $deviceCompliance.Compliant) { $riskScore += 40 }
if ($user.LastLogonTime -lt (Get-Date).AddDays(-30)) { $riskScore += 20 }

# 3. Conditional access
if ($riskScore -gt 70) {
    # Require additional verification
    Send-MFAChallenge -User $user -Method "SMS"
} elseif ($riskScore -gt 50) {
    # Allow with restrictions
    Grant-Access -User $user -Restrictions "Limited"
} else {
    # Full access
    Grant-Access -User $user -Restrictions "None"
}

# 4. Continuous monitoring
Start-Job -ScriptBlock {
    while ($true) {
        $sessions = Get-ActiveSessions
        foreach ($session in $sessions) {
            $behavior = Analyze-UserBehavior -Session $session
            if ($behavior.RiskLevel -eq "High") {
                Revoke-Session -SessionId $session.Id
                Send-Alert -Message "High risk behavior detected"
            }
        }
        Start-Sleep -Seconds 60
    }
}

🔍 Threat Hunting

Amaç: Proaktif tehdit arama ve analiz

Tehdit Avcılığı Teknikleri:

  • IOC (Indicators of Compromise): Zararlı aktivite göstergeleri
  • Behavioral Analysis: Anormal kullanıcı davranışları
  • Network Traffic Analysis: Şüpheli ağ trafiği
  • Endpoint Forensics: Sistem dosya analizi
  • Log Correlation: Çoklu kaynak log analizi
# Threat hunting queries
# 1. Suspicious PowerShell activity
Get-WinEvent -FilterHashtable @{
    LogName = 'Microsoft-Windows-PowerShell/Operational'
    ID = 4104
} | Where-Object {
    $_.Message -like "*Invoke-Expression*" -or
    $_.Message -like "*IEX*" -or
    $_.Message -like "*DownloadString*"
}

# 2. Lateral movement detection
Get-WinEvent -FilterHashtable @{
    LogName = 'Security'
    ID = 4624
} | Where-Object {
    $_.Message -like "*Logon Type: 3*" -and
    $_.Message -like "*Source Network Address: 192.168.*"
}

# 3. Privilege escalation attempts
Get-WinEvent -FilterHashtable @{
    LogName = 'Security'
    ID = 4672
} | Where-Object {
    $_.Message -like "*Privileges: SeDebugPrivilege*" -or
    $_.Message -like "*Privileges: SeBackupPrivilege*"
}

# 4. Data exfiltration patterns
Get-WinEvent -FilterHashtable @{
    LogName = 'Microsoft-Windows-Sysmon/Operational'
    ID = 3
} | Where-Object {
    $_.Message -like "*DestinationPort: 443*" -and
    $_.Message -like "*ProcessName: powershell.exe*"
}

# 5. Anomaly detection
$baseline = Get-UserBaseline -User $username -Days 30
$current = Get-UserActivity -User $username -Days 1
$anomalies = Compare-Activity -Baseline $baseline -Current $current
if ($anomalies.Count -gt 5) {
    Send-Alert -Message "Anomalous activity detected for user: $username"
}

📋 Olay Müdahale Aşamaları

1. Hazırlık (Preparation)

  • Yedekleme stratejileri

2. Tespit ve Analiz (Detection & Analysis)

  • Olay tespit sistemleri
  • Log analizi
  • Tehdit istihbaratı
  • Etki değerlendirmesi

3. Sınırlama (Containment)

  • Etkilenen sistemleri izole etme
  • Zararlı yazılımı temizleme
  • Güvenlik açıklarını kapatma
  • Geçici çözümler uygulama

4. Yok Etme (Eradication)

  • Kök neden analizi
  • Kalıcı çözümler
  • Sistem güncellemeleri
  • Güvenlik politikaları güncelleme

5. İyileştirme (Recovery)

  • Sistemleri normale döndürme
  • İş süreçlerini yeniden başlatma
  • Performans izleme
  • Kullanıcı eğitimi

6. Dersler (Lessons Learned)

  • Olay raporu hazırlama
  • İyileştirme önerileri
  • Eğitim ihtiyaçları
  • Süreç güncellemeleri
