🔒 Güvenlik & AppLocker

🎯 Windows Güvenlik Neden Kritik?

Windows güvenlik, modern işletmelerin hayatta kalması için zorunludur. Neden Windows güvenlik önemli?

🔍 Windows Güvenliğin Faydaları:

  • Veri Koruması: Hassas bilgileri koruma
  • İş Sürekliliği: Saldırılardan sonra hızla toparlanma
  • Maliyet Tasarrufu: Saldırı maliyeti milyonlarca lira
  • Müşteri Güveni: Güvenli sistem = güvenilir şirket
  • Yasal Uyumluluk: GDPR, KVKK gibi yasal gereklilikler

🏢 Gerçek Hayat Senaryosu: Ransomware Saldırısı

Saldırı: E-posta eki ile gelen ransomware

PowerShell ile Defender Yönetimi - Pratik Örnekler

🏢 Senaryo 1: Güvenlik Taraması

Durum: Şüpheli dosya tespit edildi, hızlı tarama gerekli

# 1. Defender durumunu kontrol et
Get-MpComputerStatus

# 2. Hızlı tarama başlat
Start-MpScan -ScanType QuickScan

# 3. Tarama sonucunu kontrol et
Get-MpThreatDetection | Where-Object {$_.DetectionTime -gt (Get-Date).AddHours(-1)}

# 4. Tehdit varsa detaylı bilgi al
Get-MpThreatDetection | Select-Object ThreatName, Severity, DetectionTime

🏢 Senaryo 2: Tehdit Analizi

Durum: Sistem yavaşladı, tehdit analizi gerekli

# 1. Tam sistem taraması
Start-MpScan -ScanType FullScan

# 2. Tehdit geçmişini analiz et
Get-MpThreatDetection | Sort-Object DetectionTime -Descending | Select-Object -First 10

# 3. Engellenen tehditleri kontrol et
Get-MpThreatDetection | Where-Object {$_.ActionSuccess -eq $true}

# 4. Real-time protection durumunu kontrol et
Get-MpPreference | Select-Object DisableRealtimeMonitoring

🏢 Senaryo 3: Güvenlik Yapılandırması

Durum: Yeni güvenlik politikaları uygulanacak

# 1. Real-time protection'ı etkinleştir
Set-MpPreference -DisableRealtimeMonitoring $false

# 2. Cloud-delivered protection'ı etkinleştir
Set-MpPreference -MAPSReporting Advanced

# 3. Behavioral monitoring'ı etkinleştir
Set-MpPreference -DisableBehaviorMonitoring $false

# 4. Güvenlik güncellemelerini kontrol et
Update-MpSignature
# Cloud-delivered protection'ı etkinleştir Set-MpPreference -MAPSReporting Advanced

🔒 AppLocker - Uygulama Kontrolü

AppLocker Nedir ve Neden Gerekli?

AppLocker nedir: Hangi uygulamaların sistemde çalışabileceğini kontrol eden güvenlik özelliği

Neden gerekli: Zararlı yazılımları engeller, sadece güvenli uygulamalara izin verir

🏢 Gerçek Hayat Örneği:

Problem: Çalışan bilinmeyen uygulama indirdi, sistem zarar gördü

Sonuç: Veri kaybı, sistem yeniden kurulumu gerekli

Çözüm: AppLocker ile sadece onaylı uygulamalara izin

AppLocker Kuralları - Pratik Senaryolar

📱 Executable Rules - EXE Dosya Kontrolü

Neden önemli: Zararlı EXE dosyalarını engeller

Gerçek Hayat Senaryosu:

Problem: Çalışan zararlı EXE dosyası indirdi

Çözüm: Sadece güvenli klasörlerden EXE çalıştırmaya izin

# Senaryo 1: Sadece Program Files'tan EXE çalıştır
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Program Files\*" -User Everyone

# Senaryo 2: Temp klasöründen EXE engelle
New-AppLockerPolicy -RuleType Path -Action Deny -Path "C:\Temp\*" -User Everyone

# Senaryo 3: Sadece imzalı uygulamalara izin ver
New-AppLockerPolicy -RuleType Publisher -Action Allow -Publisher "Microsoft Corporation" -User Everyone

📜 Script Rules - Script Dosya Kontrolü

Neden önemli: Zararlı scriptleri engeller

Gerçek Hayat Senaryosu:

Problem: Çalışan zararlı PowerShell scripti çalıştırdı

Çözüm: Sadece IT ekibine script çalıştırma izni

# Senaryo 1: IT ekibine script izni
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Scripts\*" -User ITAdmins

# Senaryo 2: Normal kullanıcılara script engelle
New-AppLockerPolicy -RuleType Path -Action Deny -Path "*.ps1" -User RegularUsers

# Senaryo 3: Sadece imzalı scriptlere izin
New-AppLockerPolicy -RuleType Publisher -Action Allow -Publisher "Microsoft Corporation" -User ITAdmins

📦 Installer Rules - Kurulum Dosya Kontrolü

Neden önemli: Yetkisiz yazılım kurulumunu engeller

Gerçek Hayat Senaryosu:

Problem: Çalışan yetkisiz yazılım kurdu

Çözüm: Sadece IT ekibine kurulum izni

# Senaryo 1: IT ekibine kurulum izni
New-AppLockerPolicy -RuleType Path -Action Allow -Path "C:\Software\*" -User ITAdmins

# Senaryo 2: Sadece imzalı installer'lara izin ver
New-AppLockerPolicy -RuleType Publisher -Action Allow -Publisher "Microsoft Corporation" -User Everyone

AppLocker Yapılandırması - Adım Adım Rehber

1. AppLocker Servisini Etkinleştir

Neden gerekli: AppLocker çalışması için servis aktif olmalı

Gerçek Hayat Senaryosu:

Problem: AppLocker kuralları çalışmıyor

Çözüm: Servisi etkinleştir ve otomatik başlat

# AppLocker servisini başlat
Start-Service -Name "AppIDSvc"

# Servisi otomatik başlat
Set-Service -Name "AppIDSvc" -StartupType Automatic

# Servis durumunu kontrol et
Get-Service -Name "AppIDSvc"

2. GPO ile AppLocker Kuralları Uygula

Neden gerekli: Tüm bilgisayarlarda aynı kurallar uygulanmalı

Gerçek Hayat Senaryosu:

Problem: Her bilgisayarda ayrı ayrı kurallar uygulamak zor

Çözüm: GPO ile merkezi yönetim

# AppLocker politikasını GPO'ya uygula
Set-AppLockerPolicy -XmlPolicy (Get-Content "C:\AppLockerPolicy.xml" -Raw)

# Mevcut kuralları görüntüle
Get-AppLockerPolicy -Effective

# Kuralları test et
Test-AppLockerPolicy -Path "C:\TestApp.exe" -User "TestUser"

3. AppLocker Kurallarını Test Et

Neden gerekli: Kuralların doğru çalıştığından emin olmak

Gerçek Hayat Senaryosu:

Problem: Kurallar çalışmıyor, kullanıcılar şikayet ediyor

Çözüm: Test ortamında kuralları doğrula

# Kuralları test et
Test-AppLockerPolicy -Path "C:\Program Files\TestApp\test.exe" -User "RegularUser"

# AppLocker olaylarını kontrol et
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/EXE and DLL" | Select-Object -First 10

# Kuralları raporla
Get-AppLockerPolicy -Effective | Export-Csv "C:\AppLockerReport.csv"

🔐 BitLocker - Disk Şifreleme

BitLocker Nedir ve Neden Gerekli?

BitLocker nedir: Tüm disk içeriğini şifreleyen güvenlik özelliği

Neden gerekli: Laptop çalınsa bile veriler okunamaz

🏢 Gerçek Hayat Örneği:

Problem: Laptop çalındı, şifrelenmemiş veriler sızdı

Sonuç: Müşteri verileri çalındı, yasal sorunlar

Çözüm: BitLocker ile disk şifreleme

BitLocker Özellikleri - Detaylı Açıklama

🔒 Disk Şifreleme

Ne yapar: Tüm disk içeriğini şifreler

Faydası: Veri çalınsa bile okunamaz

🛡️ TPM Desteği

Ne yapar: Donanım seviyesinde güvenlik

Faydası: Şifre kırılması çok zor

🔑 Recovery Key

Ne yapar: Şifre kurtarma anahtarı

Faydası: Şifre unutulsa bile kurtarma

⚡ Auto-unlock

Ne yapar: Otomatik şifre çözme

Faydası: Kullanıcı deneyimi kolaylaşır

BitLocker Yapılandırması - Pratik Senaryolar

🏢 Senaryo 1: Yeni Laptop Şifreleme

Durum: Yeni laptop alındı, şifreleme gerekli

# 1. BitLocker'ı etkinleştir
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -UsedSpaceOnly

# 2. Recovery key'i kaydet
$recoveryKey = (Get-BitLockerVolume -MountPoint "C:").KeyProtector | Where-Object {$_.KeyProtectorType -eq "RecoveryPassword"}
$recoveryKey.RecoveryPassword | Out-File "C:\BitLockerRecoveryKey.txt"

# 3. Şifreleme durumunu kontrol et
Get-BitLockerVolume

🏢 Senaryo 2: Şifreleme Durumu Kontrolü

Durum: Tüm bilgisayarlarda şifreleme kontrolü

# 1. Tüm disklerin şifreleme durumunu kontrol et
Get-BitLockerVolume | Select-Object MountPoint, VolumeStatus, EncryptionPercentage

# 2. Şifrelenmemiş diskleri bul
Get-BitLockerVolume | Where-Object {$_.VolumeStatus -ne "FullyEncrypted"}

# 3. Recovery key'leri kontrol et
Get-BitLockerVolume | ForEach-Object {
    $_.KeyProtector | Where-Object {$_.KeyProtectorType -eq "RecoveryPassword"}
}

🏢 Senaryo 3: Şifreleme Devre Dışı Bırakma

Durum: Bilgisayar değiştirilecek, şifreleme kaldırılacak

# 1. Şifrelemeyi devre dışı bırak
Disable-BitLocker -MountPoint "C:"

# 2. Şifreleme kaldırma durumunu kontrol et
Get-BitLockerVolume

# 3. Recovery key'leri temizle
Remove-BitLockerKeyProtector -MountPoint "C:" -KeyProtectorId "RecoveryPassword"

🔥 Windows Firewall - Ağ Güvenliği

Windows Firewall Nedir ve Neden Gerekli?

Windows Firewall nedir: Ağ trafiğini kontrol eden güvenlik duvarı

Neden gerekli: Zararlı ağ trafiğini engeller, sadece güvenli bağlantılara izin verir

🏢 Gerçek Hayat Örneği:

Problem: Hacker açık porttan sisteme sızdı

Sonuç: Veri çalındı, sistem zarar gördü

Çözüm: Windows Firewall ile port kontrolü

Firewall Kuralları - Pratik Senaryolar

📥 Inbound Rules - Gelen Trafik Kontrolü

Neden önemli: Dışarıdan gelen zararlı trafiği engeller

Gerçek Hayat Senaryosu:

Problem: Hacker RDP portuna saldırı yapıyor

Çözüm: Sadece güvenli IP'lerden RDP'ye izin ver

# Senaryo 1: HTTP trafiğine izin ver
New-NetFirewallRule -DisplayName "Allow HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow

# Senaryo 2: RDP'yi sadece IT IP'lerinden kabul et
New-NetFirewallRule -DisplayName "Allow RDP from IT" -Direction Inbound -Protocol TCP -LocalPort 3389 -RemoteAddress "192.168.1.0/24" -Action Allow

# Senaryo 3: Tüm gelen trafiği engelle (güvenlik için)
New-NetFirewallRule -DisplayName "Block All Inbound" -Direction Inbound -Action Block

📤 Outbound Rules - Giden Trafik Kontrolü

Neden önemli: Zararlı yazılımların dışarı bağlantı kurmasını engeller

Gerçek Hayat Senaryosu:

Problem: Zararlı yazılım dışarı veri gönderiyor

Çözüm: Sadece gerekli trafiğe izin ver

# Senaryo 1: HTTPS trafiğine izin ver
New-NetFirewallRule -DisplayName "Allow HTTPS" -Direction Outbound -Protocol TCP -RemotePort 443 -Action Allow

# Senaryo 2: DNS sorgularına izin ver
New-NetFirewallRule -DisplayName "Allow DNS" -Direction Outbound -Protocol UDP -RemotePort 53 -Action Allow

# Senaryo 3: Sosyal medya sitelerini engelle
New-NetFirewallRule -DisplayName "Block Social Media" -Direction Outbound -RemoteAddress "facebook.com", "twitter.com" -Action Block

Firewall Yönetimi - Pratik Komutlar

🏢 Senaryo 1: Firewall Durumu Kontrolü

Durum: Firewall çalışıyor mu kontrol et

# Firewall durumunu kontrol et
Get-NetFirewallProfile | Select-Object Name, Enabled

# Aktif kuralları listele
Get-NetFirewallRule | Where-Object {$_.Enabled -eq "True"} | Select-Object DisplayName, Direction, Action

# Belirli portu kontrol et
Get-NetFirewallRule | Where-Object {$_.DisplayName -like "*RDP*"}

🏢 Senaryo 2: Firewall Kuralı Ekleme

Durum: Yeni uygulama için kural ekle

# Yeni uygulama için kural ekle
New-NetFirewallRule -DisplayName "Allow NewApp" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow

# Kuralı test et
Test-NetConnection -ComputerName "localhost" -Port 8080

# Kuralı devre dışı bırak
Disable-NetFirewallRule -DisplayName "Allow NewApp"

# Zero Trust implementation
# 1. Identity verification
$user = Get-ADUser -Identity $username
$mfaStatus = Get-MFAStatus -User $user
$deviceCompliance = Get-DeviceCompliance -Device $deviceId

# 2. Risk assessment
$riskScore = 0
if (-not $mfaStatus.Enabled) { $riskScore += 30 }
if (-not $deviceCompliance.Compliant) { $riskScore += 40 }
if ($user.LastLogonTime -lt (Get-Date).AddDays(-30)) { $riskScore += 20 }

# 3. Conditional access
if ($riskScore -gt 70) {
    # Require additional verification
    Send-MFAChallenge -User $user -Method "SMS"
} elseif ($riskScore -gt 50) {
    # Allow with restrictions
    Grant-Access -User $user -Restrictions "Limited"
} else {
    # Full access
    Grant-Access -User $user -Restrictions "None"
}

# 4. Continuous monitoring
Start-Job -ScriptBlock {
    while ($true) {
        $sessions = Get-ActiveSessions
        foreach ($session in $sessions) {
            $behavior = Analyze-UserBehavior -Session $session
            if ($behavior.RiskLevel -eq "High") {
                Revoke-Session -SessionId $session.Id
                Send-Alert -Message "High risk behavior detected"
            }
        }
        Start-Sleep -Seconds 60
    }
}
