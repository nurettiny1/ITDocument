🔢 IP Adresleme - Basit Açıklama

IPv4 Adres Yapısı

IPv4 adresi 32 bit uzunluğunda, 4 oktetten oluşur:

192.168.1.1

Her oktet 0-255 arası değer alabilir

IP Adres Sınıfları

Class A (1.0.0.0 - 126.255.255.255)

  • Network: İlk oktet
  • Host: Son 3 oktet
  • Subnet Mask: 255.0.0.0 (/8)
  • Örnek: 10.0.0.0/8

Class B (128.0.0.0 - 191.255.255.255)

  • Network: İlk 2 oktet
  • Host: Son 2 oktet
  • Subnet Mask: 255.255.0.0 (/16)
  • Örnek: 172.16.0.0/16

Class C (192.0.0.0 - 223.255.255.255)

  • Network: İlk 3 oktet
  • Host: Son oktet
  • Subnet Mask: 255.255.255.0 (/24)
  • Örnek: 192.168.1.0/24

Subnetting Örnekleri

Örnek 1: /24 Network'ü /26'ya Bölme

Network: 192.168.1.0/24

Subnet Mask: 255.255.255.192 (/26)

Subnet'ler:

  • 192.168.1.0/26 (Host: 192.168.1.1 - 192.168.1.62)
  • 192.168.1.64/26 (Host: 192.168.1.65 - 192.168.1.126)
  • 192.168.1.128/26 (Host: 192.168.1.129 - 192.168.1.190)
  • 192.168.1.192/26 (Host: 192.168.1.193 - 192.168.1.254)

📡 Ağ Protokolleri

TCP (Transmission Control Protocol)

Özellikler:

  • Bağlantılı: 3-way handshake ile bağlantı kurar
  • Güvenilir: Veri kaybını önler, yeniden gönderir
  • Sıralı: Verileri doğru sırada iletir
  • Akış Kontrolü: Alıcının hızına göre gönderir

Kullanım Alanları:

  • Web browsing (HTTP/HTTPS)
  • E-posta (SMTP)
  • Dosya transferi (FTP)
  • Uzaktan erişim (RDP, SSH)

UDP (User Datagram Protocol)

Özellikler:

  • Bağlantısız: Handshake yapmaz
  • Hızlı: Düşük gecikme
  • Basit: Az overhead
  • Güvenilmez: Veri kaybı olabilir

Kullanım Alanları:

  • Video streaming
  • Online gaming
  • DNS sorguları
  • DHCP

🔧 Ağ Cihazları

🔄 Hub

Katman: Physical (Layer 1)

Görev: Tüm portlara sinyal gönderir

Avantaj: Basit, ucuz

Dezavantaj: Collision domain oluşturur

🔀 Switch

Katman: Data Link (Layer 2)

Görev: MAC adresine göre yönlendirir

Avantaj: Collision domain'i böler

Özellik: MAC tablosu tutar

🛣️ Router

Katman: Network (Layer 3)

Görev: IP adresine göre yönlendirir

Avantaj: Farklı network'leri bağlar

Özellik: Routing table kullanır

🔥 Firewall

Katman: Network/Application

Görev: Güvenlik filtreleme

Avantaj: Tehditleri engeller

Özellik: Policy tabanlı filtreleme

🔍 Ağ Sorun Giderme

Yaygın Ağ Sorunları

1. Bağlantı Sorunu

Semptom: İnternete erişim yok

Çözüm Adımları:

  1. Fiziksel bağlantıyı kontrol et
  2. IP adresini kontrol et (ipconfig)
  3. Gateway'e ping at
  4. DNS ayarlarını kontrol et

2. Yavaş Ağ Performansı

Semptom: Yavaş dosya transferi

Çözüm Adımları:

  1. Bant genişliğini ölç
  2. Collision'ları kontrol et
  3. Switch portlarını kontrol et
  4. Network utilization'ı analiz et

3. DNS Sorunları

Semptom: Web sitelerine erişim yok

Çözüm Adımları:

  1. DNS server ayarlarını kontrol et
  2. nslookup ile DNS sorgusu yap
  3. DNS cache'i temizle
  4. Alternatif DNS server kullan

Faydalı Komutlar

Windows Komutları

# IP yapılandırmasını görüntüle
ipconfig /all

# DNS cache'i temizle
ipconfig /flushdns

# Ağ bağlantısını test et
ping google.com

# Routing tablosunu görüntüle
route print

# ARP tablosunu görüntüle
arp -a

# Port bağlantılarını kontrol et
netstat -an

Linux Komutları

# Ağ arayüzlerini görüntüle
ip addr show

# Routing tablosunu görüntüle
ip route show

# DNS sorgusu yap
nslookup google.com

# Port dinleme durumunu kontrol et
netstat -tulpn

# Ağ trafiğini analiz et
tcpdump -i eth0

🔧 Ağ Troubleshooting Rehberi

❌ Yaygın Ağ Sorunları ve Çözümleri

Problem 1: Bağlantı Yok (No Connectivity)

Semptomlar: İnternet erişimi yok, ping çalışmıyor

Olası Nedenler:

  • Fiziksel bağlantı sorunu
  • IP adresi yapılandırma hatası
  • DNS çözümleme sorunu
  • Gateway erişim sorunu
Çözüm Adımları:
# 1. Fiziksel bağlantıyı kontrol et
ip link show
ethtool eth0

# 2. IP yapılandırmasını kontrol et
ip addr show
ip route show

# 3. DNS çözümlemesini test et
nslookup google.com
dig google.com

# 4. Gateway erişimini test et
ping 192.168.1.1
traceroute 8.8.8.8

# 5. Ağ servislerini yeniden başlat
systemctl restart networking
systemctl restart NetworkManager

Problem 2: Yavaş Ağ Performansı

Semptomlar: Düşük hız, yüksek latency

Olası Nedenler:

  • Bant genişliği sınırlaması
  • Network congestion
  • Eski network equipment
  • Malware/security issues
Çözüm Adımları:
# 1. Bant genişliğini test et
speedtest-cli
iperf3 -c server.example.com

# 2. Network trafiğini analiz et
iftop
nethogs
netstat -i

# 3. Latency'yi test et
ping -c 10 8.8.8.8
mtr 8.8.8.8

# 4. Network utilization'ı kontrol et
sar -n DEV 1 10
cat /proc/net/dev

# 5. QoS ayarlarını kontrol et
tc qdisc show
tc class show

Problem 3: Intermittent Connectivity

Semptomlar: Aralıklı bağlantı kesintileri

Olası Nedenler:

  • Fiziksel kablolama sorunu
  • Switch port sorunu
  • DHCP lease sorunu
  • Wireless interference
Çözüm Adımları:
# 1. Fiziksel bağlantıyı kontrol et
ethtool -S eth0
ethtool -i eth0

# 2. Switch port durumunu kontrol et
show interface status
show interface counters

# 3. DHCP durumunu kontrol et
dhclient -v
cat /var/log/syslog | grep dhcp

# 4. Wireless interference kontrol et
iwlist scan
iwconfig wlan0

# 5. Network logs'u analiz et
journalctl -u NetworkManager
tail -f /var/log/syslog
