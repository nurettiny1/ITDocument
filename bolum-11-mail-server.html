📧 Mail Server Yönetimi

🎯 Mail Server Neden Kritik?

Mail Server, modern işletmelerin iletişim omurgasıdır. Neden mail server önemli?

🔍 Mail Server'ın Faydaları:

  • İletişim Merkezi: Tüm iş iletişimi tek platformda
  • Güvenlik: Hassas bilgilerin güvenli iletimi
  • Verimlilik: Hızlı ve güvenilir mesajlaşma
  • Arşivleme: Önemli e-postaların saklanması
  • Maliyet Tasarrufu: Dış servis maliyetlerinden kaçınma

🏢 Gerçek Hayat Senaryosu:

Problem: 500 çalışanlı şirkette e-posta sistemi çöktü

Sonuç: 2 günlük iletişim kesintisi, iş kaybı

Çözüm: Güvenilir mail server altyapısı

🔒 Senaryo 2: E-posta Güvenliği ve Anti-Spam

Problem: Spam e-postalar ve güvenlik tehditleri

Çözüm: Kapsamlı güvenlik politikaları

Sonuç: %95 spam engelleme, %99 güvenlik

Güvenlik Yapılandırması:

# E-posta güvenliği yapılandırması
# 1. Anti-spam ayarları
Set-ContentFilterConfig -Enabled $true `
    -QuarantineRetentionPeriod 15 `
    -SpamQuarantineRetentionPeriod 15 `
    -SclQuarantineThreshold 6

# 2. IP Allow List
Add-IPAllowListEntry -IPAddress "203.0.113.0/24" -Comment "Trusted Partner Network"
Add-IPAllowListEntry -IPAddress "198.51.100.0/24" -Comment "Business Partner"

# 3. IP Block List
Add-IPBlockListEntry -IPAddress "192.0.2.0/24" -Comment "Known Spam Source"
Add-IPBlockListEntry -IPAddress "198.18.0.0/15" -Comment "Test Network"

# 4. Sender reputation
Set-SenderReputationConfig -Enabled $true -SrlBlockThreshold 7

# 5. Transport rules
New-TransportRule -Name "Block Executable Attachments" `
    -FromScope "NotInOrganization" `
    -AttachmentHasExecutableContent $true `
    -RejectMessageReasonText "Executable attachments are not allowed from external senders"

# 6. DLP politikası
New-DlpPolicy -Name "Financial Data Protection" `
    -Mode "Enforce" `
    -State "Enabled"

# 7. DLP kuralı
New-TransportRule -Name "Credit Card Detection" `
    -DlpPolicy "Financial Data Protection" `
    -MessageContainsDataClassification @{
        Name = "Credit Card Number"
        MinCount = 1
    } `
    -BlockAccess $true `
    -RejectMessageReasonText "Credit card information is not allowed in emails"

📊 Senaryo 3: Mail Server İzleme ve Performans

Problem: Mail server performans sorunları

Sonuç: %50 performans artışı, %99 uptime

İzleme ve Performans Optimizasyonu:

# Mail server izleme ve performans
# 1. Performans sayaçları
Get-Counter -Counter "\MSExchange Database\Log Generation Checkpoint Depth" `
    -SampleInterval 1 -MaxSamples 5

# 2. Mailbox veritabanı durumu
Get-MailboxDatabase -Status | Select-Object Name, DatabaseSize, AvailableNewMailboxSpace

# 3. E-posta kuyruğu durumu
Get-Queue | Select-Object Identity, MessageCount, Status

# 4. Servis durumu
Get-Service -Name "MSExchange*" | Select-Object Name, Status

# 5. Disk alanı kontrolü
Get-WmiObject -Class Win32_LogicalDisk | Where-Object {$_.DriveType -eq 3} | 
    Select-Object DeviceID, @{Name="Size(GB)";Expression={[math]::Round($_.Size/1GB,2)}}, 
    @{Name="FreeSpace(GB)";Expression={[math]::Round($_.FreeSpace/1GB,2)}}

# 6. Memory kullanımı
Get-Process -Name "MSExchange*" | Select-Object Name, WorkingSet, VirtualMemorySize

# 7. Log dosyalarını temizleme
Get-ChildItem -Path "C:\Program Files\Microsoft\Exchange Server\V15\Logging" -Recurse | 
    Where-Object {$_.LastWriteTime -lt (Get-Date).AddDays(-7)} | Remove-Item -Force

# 8. Database defragmentation
Get-MailboxDatabase | ForEach-Object {
    $db = $_.Name
    Write-Host "Defragmenting database: $db"

📡 E-posta Protokolleri

📤 SMTP (Simple Mail Transfer Protocol)

E-posta gönderme protokolü

  • Port: 25 (standart), 587 (TLS), 465 (SSL)
  • Görev: E-posta sunucuları arası iletişim
  • Kullanım: Outbound mail transfer
  • Güvenlik: STARTTLS, SSL/TLS

📥 POP3 (Post Office Protocol)

E-posta indirme protokolü

  • Port: 110 (standart), 995 (SSL)
  • Görev: E-postaları sunucudan indirme
  • Kullanım: Offline e-posta erişimi
  • Özellik: E-postalar sunucudan silinir

📨 IMAP (Internet Message Access Protocol)

E-posta senkronizasyon protokolü

  • Port: 143 (standart), 993 (SSL)
  • Görev: E-postaları sunucuda tutma
  • Kullanım: Multi-device erişim
  • Özellik: E-postalar sunucuda kalır

🌐 HTTP/HTTPS

Web tabanlı e-posta erişimi

  • Port: 80 (HTTP), 443 (HTTPS)
  • Görev: Web arayüzü üzerinden erişim
  • Kullanım: Outlook Web Access, OWA
  • Avantaj: Herhangi bir cihazdan erişim

📁 Mailbox Yönetimi

Mailbox Oluşturma

# Exchange Management Shell'de mailbox oluşturma
New-Mailbox -UserPrincipalName "john.doe@company.com" `
    -Alias "john.doe" `
    -Name "John Doe" `
    -DisplayName "John Doe" `
    -Password (ConvertTo-SecureString "TempPass123!" -AsPlainText -Force) `
    -Database "Mailbox Database 1"

# Mevcut kullanıcı için mailbox oluşturma
Enable-Mailbox -Identity "john.doe@company.com" `
    -Database "Mailbox Database 1"

# Mailbox özelliklerini güncelleme
Set-Mailbox -Identity "john.doe@company.com" `
    -ProhibitSendQuota 2GB `
    -ProhibitSendReceiveQuota 2.5GB `
    -IssueWarningQuota 1.5GB `
    -RetainDeletedItemsFor 30.00:00:00

Distribution Group Yönetimi

# Distribution group oluşturma
New-DistributionGroup -Name "IT Department" `
    -DisplayName "IT Department" `
    -Alias "IT-Dept" `
    -ManagedBy "admin@company.com" `
    -MemberJoinRestriction "Closed" `
    -MemberDepartRestriction "Closed"

# Distribution group'a üye ekleme
Add-DistributionGroupMember -Identity "IT Department" `
    -Member "john.doe@company.com"

# Distribution group üyelerini listeleme
Get-DistributionGroupMember -Identity "IT Department"

# Distribution group ayarları
Set-DistributionGroup -Identity "IT Department" `
    -RequireSenderAuthenticationEnabled $true `
    -AcceptMessagesOnlyFromSendersOrMembers "admin@company.com"

📤 Mail Flow Yönetimi

Transport Rules

# Mail flow kuralı oluşturma
New-TransportRule -Name "Block External Attachments" `
    -FromScope "NotInOrganization" `
    -AttachmentHasExecutableContent $true `
    -RejectMessageReasonText "Executable attachments are not allowed from external senders"

# Spam filtreleme kuralı
New-TransportRule -Name "High Confidence Spam" `
    -SetHeaderName "X-Spam-Level" `
    -SetHeaderValue "High" `
    -SclValue 5 `
    -Quarantine $true

# E-posta yönlendirme kuralı
New-TransportRule -Name "Forward CEO Emails" `
    -From "ceo@company.com" `
    -RedirectMessageTo "assistant@company.com"

Connector Yapılandırması

# Send connector oluşturma
New-SendConnector -Name "Internet Send Connector" `
    -AddressSpaces "SMTP:*;1" `
    -DNSRoutingEnabled $true `
    -UseExternalDNSServersEnabled $true

# Receive connector oluşturma
New-ReceiveConnector -Name "Internet Receive Connector" `
    -Usage "Internet" `
    -Bindings "0.0.0.0:25" `
    -RemoteIPRanges "0.0.0.0-255.255.255.255" `
    -AuthMechanism "Tls, BasicAuth, BasicAuthRequireTLS"
