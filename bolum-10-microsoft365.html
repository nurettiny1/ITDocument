🎯 Microsoft 365 Neden Kritik?

Microsoft 365, modern işletmelerin dijital dönüşümünün anahtarıdır. Neden Microsoft 365 kullanmalıyız?

🔍 Microsoft 365'in Faydaları:

  • Bulut Avantajları: Her yerden erişim, otomatik güncellemeler
  • Maliyet Tasarrufu: On-premises sunucu maliyetleri yok
  • İşbirliği: Ekip çalışması için gelişmiş araçlar
  • Güvenlik: Enterprise seviye güvenlik özellikleri

Adım Adım Kullanıcı Oluşturma:
# 1. Microsoft 365'e bağlan
Connect-MsolService

# 2. Yeni kullanıcı oluştur
New-MsolUser -UserPrincipalName "john.doe@company.com" `
    -DisplayName "John Doe" `
    -FirstName "John" `
    -LastName "Doe" `
    -Password "TempPass123!" `
    -ForceChangePassword $true

# 3. Lisans ata
Set-MsolUserLicense -UserPrincipalName "john.doe@company.com" `
    -AddLicenses "company:ENTERPRISEPACK"

# 4. Grup üyeliği ekle
Add-MsolGroupMember -GroupObjectId "group-guid" `
    -GroupMemberType "User" `
    -GroupMemberObjectId "user-guid"

🏢 Senaryo 2: Toplu Kullanıcı Yönetimi

Problem: 200 çalışan için toplu hesap oluşturma

Çözüm: CSV dosyasından otomatik kullanıcı oluşturma

Sonuç: 1 hafta yerine 1 gün, %95 başarı

Toplu Kullanıcı Oluşturma Scripti:
# CSV dosyasından toplu kullanıcı oluşturma
$users = Import-Csv -Path "C:\Users\new_users.csv"

foreach ($user in $users) {
    try {
        # Kullanıcı oluştur
        New-MsolUser -UserPrincipalName $user.Email `
            -DisplayName $user.FullName `
            -FirstName $user.FirstName `
            -LastName $user.LastName `
            -Password "TempPass123!" `
            -ForceChangePassword $true
        
        # Lisans ata
        Set-MsolUserLicense -UserPrincipalName $user.Email `
            -AddLicenses "company:ENTERPRISEPACK"
        
        # Grup üyeliği ekle
        Add-MsolGroupMember -GroupObjectId $user.GroupId `
            -GroupMemberType "User" `
            -GroupMemberObjectId (Get-MsolUser -UserPrincipalName $user.Email).ObjectId
        
        Write-Host "✅ Kullanıcı oluşturuldu: $($user.FullName)" -ForegroundColor Green
    }
    catch {
        Write-Host "❌ Hata: $($user.FullName) - $($_.Exception.Message)" -ForegroundColor Red
    }
}

🏢 Senaryo 3: Kullanıcı Grup Yönetimi

Problem: Departman bazlı grup yönetimi karmaşık

Çözüm: Otomatik grup üyeliği yönetimi

Sonuç: %80 otomasyon, %90 doğruluk

Grup Yönetimi Scripti:
# Departman bazlı grup yönetimi
$departments = @{
    "IT" = "IT-Users"
    "HR" = "HR-Users"
    "Finance" = "Finance-Users"
    "Sales" = "Sales-Users"
}

foreach ($dept in $departments.Keys) {
    # Departman kullanıcılarını al
    $users = Get-MsolUser | Where-Object {$_.Department -eq $dept}
    
    foreach ($user in $users) {
        # Grup üyeliği ekle
        Add-MsolGroupMember -GroupObjectId (Get-MsolGroup -SearchString $departments[$dept]).ObjectId `
            -GroupMemberType "User" `
            -GroupMemberObjectId $user.ObjectId
        
        Write-Host "✅ $($user.DisplayName) -> $($departments[$dept])" -ForegroundColor Green
    }
}

Güvenlik Yapılandırması:
# 1. Exchange Online'a bağlan
Connect-ExchangeOnline

# 2. Spam filtreleme kuralı
New-TransportRule -Name "High Confidence Spam" `
    -SetHeaderName "X-Spam-Level" `
    -SetHeaderValue "High" `
    -SclValue 5 `
    -Quarantine $true

# 3. Kötü amaçlı ek engelleme
New-TransportRule -Name "Block Malicious Attachments" `
    -FromScope "NotInOrganization" `
    -AttachmentHasExecutableContent $true `
    -RejectMessageReasonText "Executable attachments are not allowed from external senders"

# 4. Hassas veri koruması
New-DlpPolicy -Name "Credit Card Protection" `
    -Template "Credit Card Number" `
    -State "Enabled"

🏢 Senaryo 2: Mailbox Yönetimi ve Optimizasyonu

Problem: Mailbox boyutları aşırı, performans düşük, arşivleme yok

Sonuç: %80 performans artışı, %90 alan tasarrufu

Mailbox Yönetimi Scripti:
# 1. Mailbox boyut limitleri ayarla
Set-Mailbox -Identity "john.doe@company.com" `
    -ProhibitSendQuota 2GB `
    -ProhibitSendReceiveQuota 2.5GB `
    -IssueWarningQuota 1.5GB

# 2. Otomatik arşivleme etkinleştir
Enable-Mailbox -Identity "john.doe@company.com" -Archive

# 3. Retention policy oluştur
New-RetentionPolicy -Name "Default Retention Policy" `
    -RetentionPolicyTagLinks "Default 2 Year Move to Archive"

# 4. Mailbox'a retention policy ata
Set-Mailbox -Identity "john.doe@company.com" `
    -RetentionPolicy "Default Retention Policy"

🏢 Senaryo 3: Distribution Group Yönetimi

Sonuç: %90 grup verimliliği, %95 otomasyon

Distribution Group Yönetimi:
# 1. Distribution group oluştur
New-DistributionGroup -Name "IT Department" `
    -DisplayName "IT Department" `
    -Alias "IT-Dept" `
    -ManagedBy "admin@company.com"

# 2. Toplu üye ekleme
$itUsers = Get-Mailbox | Where-Object {$_.Department -eq "IT"}
foreach ($user in $itUsers) {
    Add-DistributionGroupMember -Identity "IT Department" `

}

# 3. Grup ayarları

Gerçek Hayat Güvenlik Yönetimi Senaryoları

🏢 Senaryo 1: Multi-Factor Authentication (MFA) Yapılandırması

Problem: Kullanıcı hesapları güvenli değil, şifre saldırıları riski

Çözüm: MFA ile çok faktörlü kimlik doğrulama

Sonuç: %99 güvenlik artışı, %95 saldırı önleme

MFA Yapılandırması:
# 1. MFA'yı etkinleştir
Set-MsolUser -UserPrincipalName "john.doe@company.com" `
    -StrongAuthenticationRequired $true

# 2. MFA durumunu kontrol et
Get-MsolUser -UserPrincipalName "john.doe@company.com" `
    | Select-Object UserPrincipalName, StrongAuthenticationRequirements

# 3. Toplu MFA etkinleştirme
$users = Get-MsolUser | Where-Object {$_.UserType -eq "Member"}
foreach ($user in $users) {
    Set-MsolUser -UserPrincipalName $user.UserPrincipalName `
        -StrongAuthenticationRequired $true
}

🏢 Senaryo 2: Conditional Access Policy Yapılandırması

Problem: Hassas verilere erişim kontrolü yok, güvenlik açıkları

Çözüm: Conditional Access ile koşullu erişim kontrolü

Sonuç: %95 erişim güvenliği, %90 risk azaltma

Conditional Access Policy:
# 1. Conditional Access politikası oluştur
New-AzureADMSConditionalAccessPolicy -DisplayName "Require MFA for Admin" `
    -State "Enabled" `
    -Conditions @{
        Users = @{
            IncludeUsers = @("admin@company.com")
        }
        Applications = @{
            IncludeApplications = @("All")
        }
        ClientAppTypes = @("Browser", "MobileAppsAndDesktopClients")
    } `
    -GrantControls @{
        BuiltInControls = @("Mfa")
    }

# 2. Policy durumunu kontrol et
Get-AzureADMSConditionalAccessPolicy

🏢 Senaryo 3: Data Loss Prevention (DLP) Yapılandırması

Sonuç: %99 veri koruması, %100 uyumluluk

DLP Policy Yapılandırması:
# 1. DLP politikası oluştur
New-DlpCompliancePolicy -Name "Financial Data Protection" `
    -Comment "Protect financial data from unauthorized sharing" `
    -Mode "Enforce"

# 2. DLP kuralı oluştur
New-DlpComplianceRule -Name "Credit Card Detection" `
    -Policy "Financial Data Protection" `
    -ContentContainsSensitiveInformation @{
        SensitiveInformation = @{
            Name = "Credit Card Number"
            MinCount = 1
        }
    } `
    -BlockAccess $true

# 3. Policy durumunu kontrol et
Get-DlpCompliancePolicy -Identity "Financial Data Protection"
