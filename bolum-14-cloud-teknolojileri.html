☁️ Cloud Teknolojileri

🎯 Cloud Teknolojileri Neden Önemli?

Cloud teknolojileri, modern işletmelerin dijital dönüşümünün anahtarıdır. Neden cloud teknolojileri önemli?

🔍 Cloud Teknolojilerinin Faydaları:

  • Maliyet Tasarrufu: On-premises sunucu maliyetleri yok
  • Ölçeklenebilirlik: İhtiyaca göre kaynak artırma/azaltma
  • Esneklik: Her yerden erişim, mobil çalışma
  • Güvenlik: Enterprise seviye güvenlik özellikleri
  • Hız: Hızlı deployment, otomatik güncellemeler

🏢 Gerçek Hayat Senaryosu:

Problem: 200 çalışanlı şirkette on-premises sistemler yetersiz

Sonuç: Yüksek maliyet, düşük performans, güvenlik açıkları

Çözüm: Cloud migration ile modern altyapı

🏗️ Bulut Servis Modelleri

🖥️ IaaS (Infrastructure as a Service)

Temel altyapı hizmetleri

  • Örnekler: Azure VMs, AWS EC2, Google Compute Engine
  • Kontrol: İşletim sistemi, uygulamalar, veriler
  • Yönetim: Sanal makineler, depolama, ağ
  • Kullanım: Geliştirme, test, production

⚙️ PaaS (Platform as a Service)

Platform hizmetleri

  • Örnekler: Azure App Service, AWS Elastic Beanstalk, Google App Engine
  • Kontrol: Uygulamalar, veriler
  • Yönetim: Runtime, middleware, database
  • Kullanım: Uygulama geliştirme, deployment

📱 SaaS (Software as a Service)

Yazılım hizmetleri

  • Örnekler: Office 365, Salesforce, Google Workspace
  • Kontrol: Sadece veriler
  • Yönetim: Uygulama, altyapı, platform
  • Kullanım: End-user uygulamaları

🌐 Bulut Dağıtım Modelleri

☁️ Public Cloud

Genel bulut hizmetleri

  • Avantajlar: Düşük maliyet, ölçeklenebilirlik
  • Dezavantajlar: Güvenlik endişeleri, kontrol sınırları
  • Örnekler: Azure, AWS, Google Cloud
  • Kullanım: Genel amaçlı uygulamalar

🏢 Private Cloud

Özel bulut hizmetleri

  • Avantajlar: Yüksek güvenlik, tam kontrol
  • Dezavantajlar: Yüksek maliyet, karmaşık yönetim
  • Örnekler: VMware vCloud, OpenStack
  • Kullanım: Kritik uygulamalar

🔀 Hybrid Cloud

Hibrit bulut çözümleri

  • Avantajlar: Esneklik, optimizasyon
  • Dezavantajlar: Karmaşık yönetim, entegrasyon
  • Örnekler: Azure Arc, AWS Outposts
  • Kullanım: Karmaşık işletmeler

🔵 Microsoft Azure

Azure Temel Servisleri

💻 Compute Services

  • Virtual Machines: IaaS sanal makineler
  • App Service: PaaS web uygulamaları
  • Azure Functions: Serverless computing
  • Container Instances: Container hosting

💾 Storage Services

  • Blob Storage: Dosya depolama
  • File Storage: SMB paylaşımları
  • Queue Storage: Mesaj kuyrukları
  • Table Storage: NoSQL veritabanı

🌐 Networking Services

  • Virtual Network: Sanal ağ
  • Load Balancer: Yük dengeleme
  • Application Gateway: Web uygulaması güvenliği
  • VPN Gateway: Site-to-site VPN

Azure PowerShell Komutları

# Azure'a bağlan
Connect-AzAccount

# Subscription'ı seç
Set-AzContext -SubscriptionId "subscription-id"

# Resource Group oluştur
New-AzResourceGroup -Name "MyResourceGroup" -Location "East US"

# Virtual Machine oluştur
New-AzVm -ResourceGroupName "MyResourceGroup" -Name "MyVM" -Location "East US" -VirtualNetworkName "MyVNet" -SubnetName "MySubnet" -SecurityGroupName "MyNSG" -PublicIpAddressName "MyPublicIP" -OpenPorts 80,3389

# Storage Account oluştur
New-AzStorageAccount -ResourceGroupName "MyResourceGroup" -Name "mystorageaccount" -Location "East US" -SkuName "Standard_LRS"

# Web App oluştur
New-AzAppServicePlan -ResourceGroupName "MyResourceGroup" -Name "MyAppServicePlan" -Location "East US" -Tier "Free"
New-AzWebApp -ResourceGroupName "MyResourceGroup" -Name "MyWebApp" -AppServicePlan "MyAppServicePlan"

🟠 Amazon Web Services (AWS)

AWS Temel Servisleri

💻 Compute Services

  • EC2: Elastic Compute Cloud
  • Lambda: Serverless computing
  • ECS: Container service
  • Elastic Beanstalk: PaaS platform

💾 Storage Services

  • S3: Simple Storage Service
  • EBS: Elastic Block Store
  • EFS: Elastic File System
  • Glacier: Archive storage

🌐 Networking Services

  • VPC: Virtual Private Cloud
  • ELB: Elastic Load Balancer
  • CloudFront: CDN service
  • Route 53: DNS service

AWS CLI Komutları

# AWS CLI yapılandırması
aws configure

# EC2 instance oluştur
aws ec2 run-instances --image-id ami-0abcdef1234567890 --count 1 --instance-type t2.micro --key-name MyKeyPair --security-group-ids sg-12345678 --subnet-id subnet-12345678

# S3 bucket oluştur
aws s3 mb s3://my-bucket-name

# S3 dosya yükle
aws s3 cp local-file.txt s3://my-bucket-name/

# Lambda function oluştur
aws lambda create-function --function-name my-function --runtime python3.8 --role arn:aws:iam::123456789012:role/lambda-role --handler index.handler --zip-file fileb://function.zip

Güvenlik En İyi Uygulamaları

# Azure Key Vault ile şifreleme
New-AzKeyVault -VaultName "MyKeyVault" -ResourceGroupName "MyResourceGroup" -Location "East US"

# Azure AD ile kimlik doğrulama
New-AzADApplication -DisplayName "MyApp" -IdentifierUris "https://myapp.com"

# AWS IAM ile erişim kontrolü
aws iam create-user --user-name MyUser
aws iam attach-user-policy --user-name MyUser --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

# GCP IAM ile erişim kontrolü
gcloud iam service-accounts create my-service-account --display-name "My Service Account"
gcloud projects add-iam-policy-binding PROJECT_ID --member="serviceAccount:my-service-account@PROJECT_ID.iam.gserviceaccount.com" --role="roles/storage.objectViewer"

🏢 Gerçek Hayat Senaryosu

Hibrit Bulut Geçiş Projesi

Şirket: 1000 çalışanlı finansal hizmetler firması

Proje: On-premises'den hibrit buluta geçiş

Geçiş Stratejisi:

📋 1. Faz - Planlama (2 ay)
  • Mevcut altyapı analizi
  • Bulut stratejisi belirleme
  • Güvenlik gereksinimleri
  • Maliyet analizi
🔧 2. Faz - Hazırlık (3 ay)
  • Azure tenant yapılandırması
  • Hibrit bağlantı kurulumu
  • Güvenlik politikaları
  • Pilot kullanıcı seçimi
🧪 3. Faz - Pilot (2 ay)
  • Pilot kullanıcıların geçişi
  • Fonksiyonel testler
  • Performans değerlendirmesi
  • Kullanıcı eğitimleri
🚀 4. Faz - Tam Geçiş (6 ay)
  • Kademeli geçiş
  • Veri migrasyonu
  • Uygulama modernizasyonu
  • Go-live desteği

💰 Bulut Maliyet Optimizasyonu

Maliyet Azaltma Stratejileri

📊 Right-Sizing

  • VM boyutlarını optimize et
  • Kullanılmayan kaynakları kapat
  • Performans metriklerini izle
  • Otomatik ölçeklendirme kullan

⏰ Reserved Instances

  • Uzun vadeli taahhütler
  • %30-70 maliyet tasarrufu
  • 1-3 yıl süreli anlaşmalar
  • Kullanım öngörüleri

🔄 Auto-Scaling

  • Otomatik ölçeklendirme
  • Demand-based pricing
  • Resource utilization
  • Cost-performance balance

Maliyet İzleme Araçları

# Azure maliyet analizi
Get-AzConsumptionUsageDetail -BillingPeriodName "2023-01" | Select-Object InstanceName, UsageStart, UsageEnd, PretaxCost

# AWS maliyet analizi
aws ce get-cost-and-usage --time-period Start=2023-01-01,End=2023-01-31 --granularity MONTHLY --metrics BlendedCost

# GCP maliyet analizi
gcloud billing budgets list --billing-account=BILLING_ACCOUNT_ID
